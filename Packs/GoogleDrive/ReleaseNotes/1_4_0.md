
#### Integrations

##### Google Drive

- Added optional **soft_delete** argument to ***google-drive-file-delete***. Default `false` preserves the existing permanent-delete behavior; when `true`, the file is moved to the user's Trash instead of being permanently deleted (reversible via the Drive UI or API).
- Added the `GoogleDrive.File.File.trashed` output to ***google-drive-file-delete*** (populated only when `soft_delete=true`).
- Added optional **transfer_ownership** argument to ***google-drive-file-permission-create***. Default `false` preserves the existing request shape; when `true`, ownership of the file is transferred to the new permission holder by appending `transferOwnership=true` to the request URL.
- Added optional **ignore_not_found** argument to ***google-drive-file-permission-delete***. Default `false` preserves the existing behavior (404 raises an error); when `true`, an HTTP 404 response (the permission is already absent) is treated as success — useful for idempotent operations. Other error codes continue to propagate as before.
- Added the following purely-additive context outputs to ***google-drive-file-permissions-list*** (populated when returned by the Drive API):
  - `GoogleDrive.FilePermission.FilePermission.permissionDetails.inherited` (Boolean)
  - `GoogleDrive.FilePermission.FilePermission.permissionDetails.permissionType` (String)
  - `GoogleDrive.FilePermission.FilePermission.permissionDetails.role` (String)
  - `GoogleDrive.FilePermission.FilePermission.permissionDetails.inheritedFrom` (String)
