#### Integrations

##### Axonius

- Added **axonius-get-assets** — a generic command using `POST /api/v2/assets/{asset_type}` supporting any asset type (devices, users, vulnerability_instances, security_findings, and more). Supports `query`, `fields`, `fields_to_exclude`, `page_limit`, `next_page`, `include_metadata`, `include_details`, and `use_cache_entry` parameters with cursor-based pagination.
- Added **axonius-get-asset-types** — returns all available asset types in the Axonius instance via `GET /api/v2/assets/asset_types`.
- Added **axonius-get-custom-data** — lists Custom Data Management entries via `GET /api/v2/custom_data_management`.
- Added **axonius-create-custom-data** — creates a Custom Data Management entry via `POST /api/v2/custom_data_management`.
- Added **axonius-delete-custom-data** — deletes a Custom Data Management entry by ID via `DELETE /api/v2/custom_data_management/{id}`.
- Added **axonius-get-enforcements** — lists enforcement sets via `GET /api/v2/enforcements`.
- Added **axonius-run-enforcement** — triggers an enforcement run via `POST /api/v2/enforcements/{enforcement_id}/run`.
- Added **axonius-get-queries** — lists saved queries via `GET /api/v2/queries` with optional asset_type filter.
- Added **axonius-create-query** — creates a saved query via `POST /api/v2/queries`.
- Added **axonius-delete-query** — deletes a saved query via `DELETE /api/v2/queries/{query_id}`.
- Added **axonius-get-grouped-vulnerabilities** — fetches all pages of vulnerability instances, groups them by CVE ID, and returns the Top N CVEs sorted by affected host count with optional `team_name` and `urgent` filters. Outputs `Axonius.GroupedVulnerabilities` keyed by `cve_id`.
- Improved error handling across all API commands: non-2xx HTTP responses now raise explicit exceptions to prevent silent Playbook failures.
- Extended `make_api_call` to support `GET` and `DELETE` HTTP methods in addition to `POST`.
