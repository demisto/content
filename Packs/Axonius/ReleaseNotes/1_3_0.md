#### Integrations
##### Axonius
- Added the ***axonius-get-assets*** command, which fetches assets of any type. Supports all asset types including *vulnerability_instances*. Use *next_token* for cursor-based pagination.
- Added the ***axonius-get-asset-types*** command, which returns all available asset types in the Axonius instance.
- Added the ***axonius-get-custom-data*** command, which lists Custom Data Management entries.
- Added the ***axonius-create-custom-data*** command, which creates a Custom Data Management entry.
- Added the ***axonius-delete-custom-data*** command, which deletes a Custom Data Management entry by ID.
- Added the ***axonius-get-enforcements*** command, which lists enforcement sets.
- Added the ***axonius-run-enforcement*** command, which triggers an enforcement run.
- Added the ***axonius-get-queries*** command, which lists saved queries with an optional *asset_type* filter.
- Added the ***axonius-create-query*** command, which creates a saved query.
- Added the ***axonius-delete-query*** command, which deletes a saved query.
- Added the ***axonius-get-grouped-vulnerabilities*** command, which fetches all vulnerability instances, groups them by CVE ID, and returns the top N CVEs sorted by affected host count. Supports optional *team_name* and *urgent* filters. Outputs *Axonius.GroupedVulnerabilities* keyed by *cve_id*.
- Improved implementation of error handling across all API commands. Non-2xx HTTP responses now raise explicit exceptions to prevent silent playbook failures.
- Improved implementation of the *make_api_call* helper to support *GET* and *DELETE* HTTP methods in addition to *POST*.
