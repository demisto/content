
#### Integrations

##### Amazon Web Services

- Added support for **aws-ec2-instance-stopped-waiter** command that waits until ec2 instances are in the 'stopped' state. checks every `waiter_delay` seconds until successful or until the maximum number of attempts (`waiter_max_attempts`) is reached.
- Added support for **aws-lambda-function-create** command that creates a lambda function. to create a function, you need a deployment package and an execution role.
- Added support for **aws-ec2-instance-status-ok-waiter** command that waits until ec2 instance status checks pass. checks every `waiter_delay` seconds until successful or until the maximum number of attempts (`waiter_max_attempts`) is reached.
- Added support for **aws-ec2-reserved-instances-describe** command that describes one or more of the reserved instances that you purchased.
- Added support for **aws-ec2-address-associate** command that associates an elastic ip address, or carrier ip address (for instances that are in subnets in wavelength zones) with an instance or a network interface.
- Added support for **aws-ec2-instances-monitor** command that enables detailed monitoring on one or more running amazon ec2 instances.
- Added support for **aws-ec2-iam-instance-profile-associations-describe** command that describes iam instance profile associations.
- Added support for **aws-lambda-function-versions-list** command that returns a list of versions, with the versionspecific configuration of each.
- Added support for **aws-ec2-addresses-describe** command that describes one or more of your elastic ip addresses.
- Added support for **aws-s3-bucket-delete** command that delete aws s3 bucket, the bucket must be empty from files.
- Added support for **aws-ec2-address-allocate** command that allocates an elastic ip address to your aws account. after you allocate the elastic ip address you can associate it with an instance or network interface.
- Added support for **aws-ec2-image-create** command that creates an amazon machine image (ami) from an amazon ebsbacked instance. the instance must be in the running or stopped state.
- Added support for **aws-lambda-function-get** command that returns information about the function or the specified version, including a link to download the deployment package (valid for 10 minutes). if a version is specified, only versionspecific details are returned.
- Added support for **aws-ec2-images-describe** command that describes the specified images (amis, akis, and aris) available to you or all of the images available to you.
- Added support for **aws-ec2-image-available-waiter** command that waits until an ami is in the 'available' state. this command polls the ami status until it becomes available or the maximum wait time is reached.
- Added support for **aws-s3-bucket-objects-list** command that returns some or all (up to 1,000) of the objects in a bucket.
- Added support for **aws-lambda-layer-version-publish** command that creates a lambda layer from a zip archive.
- Added support for **aws-lambda-function-url-config-delete** command that deletes a lambda function url. when you delete a function url, you can't recover it. creating a new function url results in a different url address.
- Added support for **aws-ec2-instance-terminated-waiter** command that waits until the specified ec2 instances reach the 'terminated' state. checks every `waiter_delay` seconds until successful or until the maximum number of attempts (`waiter_max_attempts`) is reached.
- Added support for **aws-ec2-address-disassociate** command that disassociates an elastic ip address from the instance or network interface it's associated with.
- Added support for **aws-ec2-address-release** command that releases the specified elastic ip address. after releasing an elastic ip address, it is released to the ip address pool and might be unavailable to you.
- Added support for **aws-lambda-layer-version-list** command that lists the versions of an lambda layer.
- Added support for **aws-ec2-instance-running-waiter** command that waits until the specified ec2 instances reach the 'running' state. checks the status every `waiter_delay` seconds until successful or until `waiter_max_attempts` is reached (default maximum attempts: `waiter_max_attempts`).
- Added support for **aws-ec2-image-copy** command that initiates the copy of an ami from the specified source region to the current region. you can copy an ami across regions to enable consistent global deployment.
- Added support for **aws-ec2-password-data-get** command that retrieves the encrypted administrator password for a running windows instance.
- Added support for **aws-lambda-aliases-list** command that returns a list of aliases created for a lambda function.
- Added support for **aws-ec2-instances-reboot** command that requests a reboot of one or more instances. this operation is asynchronous; it only queues a request to reboot the specified instances.
- Added support for **aws-lambda-layer-version-delete** command that deletes a version of a lambda layer.
- Added support for **aws-lambda-functions-list** command that returns a list of your lambda functions. for each function, the response includes the function configuration information.
- Added support for **aws-ec2-instances-unmonitor** command that disables detailed monitoring for one or more running amazon ec2 instances.
- Added support for **aws-lambda-function-delete** command that deletes a lambda function.
- Added support for **aws-ec2-image-deregister** command that deregisters the specified amazon machine image (ami). after you deregister an ami, it can't be used to launch new instances. however, it doesn't affect any instances that you've already launched from the ami.
- Added support for **aws-lambda-account-settings-get** command that retrieves details about the account's limits and usage in an aws region.
- Updated the **aws-rds-db-instance-modify** command to not use the *vpc_security_group_ids* argument.
