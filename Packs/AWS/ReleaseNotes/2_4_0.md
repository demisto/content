
#### Integrations

##### Amazon Web Services

- Added support for installing the **AWS** content pack in the **Cortex XSOAR** and **Cortex XSIAM** Marketplaces (in addition to the Cortex Cloud platform).
- Added the following configuration parameters used when running the integration outside the Cortex Cloud connector flow:
  - *Access Key* (paired with *Secret Key*) - the long-lived AWS Access Key ID and Secret Access Key.
  - *Role ARN* - the full ARN of the role to assume via AWS STS.
  - *STS Region* - the region for AWS STS calls.
  - *AWS organization accounts* - a comma-separated list of AWS organization accounts to use when running commands across multiple accounts.
  - *Max concurrent command calls* - concurrency cap for multi-account commands.
- Updated the authentication flow to resolve AWS credentials from the integration parameters and automatically invoke the STS ***AssumeRole*** command when a *Role ARN* is provided in Cortex XSOAR or Cortex XSIAM.
- Added support for *Timeout* and *Retries* integration parameters on every AWS service client. The timeout value is parsed as ``"<read>,<connect>"`` (or just ``"<read>"`` with a default connect timeout of 10 seconds) and retries is capped at 10.
- Updated the **AWS STS Regional Endpoints** integration parameter to wire through the *AWS_STS_REGIONAL_ENDPOINTS* environment variable so botocore picks the configured (*legacy* or `*regional*) STS endpoint resolution mode.
- Added support to run commands on multiple accounts. When *Role name for cross-organization account access* and *AWS organization accounts* are both configured, commands are executed in parallel across every listed account using *ThreadPoolExecutor* (mirroring the pattern from the legacy **AWS-EC2** content pack). Each per-account result is tagged with its *AccountId*, and failures from one account do not abort the rest of the batch.
- Updated the *Test* functionality to perform a live `sts:GetCallerIdentity` call.
- Updated the integration to remove the redundant *Main Role* parameter. The *Role name* parameter is now the single field used for role-name-based authentication across accounts.
- Updated the integration to remove the redundant *Default AWS Access Key ID* and *Default AWS Secret Access Key* parameters. The paired *Access Key* / *Secret Key* parameter is now the single source for AWS access credentials in Cortex XSOAR and Cortex XSIAM. The Cortex Cloud connector flow is unaffected.
- Removed the unused *Default AWS Account ID* (`test_account_id`) configuration parameter.
- Fixed an issue where ***get_service_client*** returned the original (unbuilt) session instead of the newly constructed boto3 session, preventing session reuse in the health check flow.
- Added the **STS** value to the AWSServices enum to support the ***sts:GetCallerIdentity*** call in the test_module.
