
#### Integrations

##### Amazon Web Services

- Added support for installing the **AWS** pack on the **Cortex XSOAR** and **Cortex XSIAM** marketplaces (in addition to the Cortex Cloud platform).
- Added the following configuration parameters used when running the integration outside the Cortex Cloud connector flow:
  - *Access Key* (paired with *Secret Key*) - long-lived AWS Access Key ID and Secret Access Key.
  - *Role ARN* - full ARN of the role to assume via AWS STS.
  - *STS Region* - explicit region for AWS STS calls.
  - *AWS organization accounts* - comma-separated list of AWS Organization accounts to use when running commands across multiple accounts.
  - *Max concurrent command calls* - concurrency cap for multi-account commands.
- Updated the authentication flow so that, when the integration is configured on **Cortex XSOAR** or **Cortex XSIAM**, AWS credentials are resolved from the integration parameters and STS `AssumeRole` is invoked automatically when a *Role ARN* is provided.
- Added support for *Timeout* and *Retries* integration parameters on every AWS service client. The ``timeout`` value is parsed as ``"<read>,<connect>"`` (or just ``"<read>"`` with a default connect timeout of 10 seconds) and ``retries`` is capped at 10.
- Updated *AWS STS Regional Endpoints* integration parameter to wire through the ``AWS_STS_REGIONAL_ENDPOINTS`` environment variable so botocore picks the configured (``legacy`` or ``regional``) STS endpoint resolution mode.
- Added multi-account command fan-out. When *Role name for cross-organization account access* and *AWS organization accounts* are both configured, commands are executed in parallel across every listed account using `ThreadPoolExecutor` (mirroring the pattern from the legacy `AWS-EC2` pack). Each per-account result is tagged with its `AccountId`, and failures from one account do not abort the rest of the batch.
- Updated *Test* to perform a live `sts:GetCallerIdentity` call when configured with marketplace credentials.
- Removed the redundant *Main Role* (`role_name`) configuration parameter; the *Role name for cross-organization account access* (`access_role_name`) parameter is the single field used for role-name-based authentication across accounts.
- Removed the redundant *Default AWS Access Key ID* (`access_key_id`) and *Default AWS Secret Access Key* (`secret_access_key`) configuration parameters. The paired *Access Key* / *Secret Key* (`credentials`) field is now the single source for AWS access credentials on **Cortex XSOAR** and **Cortex XSIAM** marketplaces. The Cortex Cloud platform connector flow is unaffected.
- Removed the unused *Default AWS Account ID* (`test_account_id`) configuration parameter.
- Fixed a bug where `get_service_client` returned the original (unbuilt) session instead of the newly constructed boto3 session, preventing session reuse in the health check flow.
- Added `STS` to the `AWSServices` enum to support the `sts:GetCallerIdentity` call in `test_module`.
- Maintained existing behavior on the Cortex Cloud platform (Cortex Cloud connector flow).
