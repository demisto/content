
#### Integrations

##### New: AWS - Security Hub v2

Added the AWS Security Hub V2 integration, enabling ingestion and management of unified security and compliance findings in OCSF format through the AWS Security Hub V2 API. The integration imports AWS Security Hub V2 findings into Cortex with configurable first fetch time, maximum findings per fetch, minimum severity, and OCSF field-based filtering. It also supports bidirectional mirroring of finding updates between AWS Security Hub V2 and Cortex XSOAR.

- Added the following commands:
  - ***aws-securityhub-v2-security-hub-enable***
  - ***aws-securityhub-v2-security-hub-disable***
  - ***aws-securityhub-v2-findings-get***
  - ***aws-securityhub-v2-findings-batch-update***
- Added support for bidirectional incident mirroring between AWS Security Hub and Cortex XSOAR (see the README for details).
- Added the ***Incident Mirroring Direction*** parameter.
- Added the ***Resolve finding of closed incident from Cortex XSOAR in AWS Security Hub*** parameter.

#### Mappers

##### New: AWS Security Hub v2 - Incoming Mapper

Added an incoming mapper that maps OCSF-formatted findings to Cortex XSOAR incident fields for incoming mirroring, including finding details, severity, status, resources, remediation, and compliance information.

##### New: AWS Security Hub v2 - Outgoing Mapper

Added an outgoing mapper that maps Cortex XSOAR incident fields (severity, status, and comment) to AWS Security Hub V2 finding fields for outgoing mirroring.
