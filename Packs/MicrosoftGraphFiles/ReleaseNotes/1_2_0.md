
#### Integrations

##### O365 File Management (Onedrive/Sharepoint/Teams)

- Added the ***msgraph-driveitem-update*** command. Issues `PATCH /v1.0/{drive-prefix}/items/{item-id}` to move (within or across drives), rename, or update metadata on a driveItem. All update fields (`new_parent_id`, `new_parent_drive_id`, `new_name`, `description`, `conflict_behavior`) are optional; at least one must be supplied.
- Added the ***msgraph-driveitem-copy*** command. Issues `POST /v1.0/{drive-prefix}/items/{item-id}/copy`. Microsoft Graph performs the copy asynchronously and responds `202 Accepted` with a monitor URL; the command returns the monitor URL under `MsGraphFiles.CopyOperation.MonitorUrl` so the caller can poll for status directly against Microsoft Graph.
- Added the ***msgraph-driveitem-permissions-list*** command. Issues `GET /v1.0/{drive-prefix}/items/{item-id}/permissions` to enumerate the sharing permissions on a driveItem. Supports pagination via `limit` and `next_page_url`. The war-room table now surfaces a unified `GrantedTo` column that walks every IdentitySet field Microsoft Graph may populate (`grantedToV2.user`/`siteUser`/`group`/`application`, `grantedTo`, and `grantedToIdentitiesV2[]`) and decodes SharePoint claims-encoded `siteUser.loginName` values (e.g., external guest users on OneDrive/SharePoint) back to a readable email.
- Added the ***msgraph-driveitem-permission-delete*** command. Issues `DELETE /v1.0/{drive-prefix}/items/{item-id}/permissions/{permission-id}` to revoke a single sharing permission.
- Extended the integration `ok_codes` tuple to include `202`, required by the new asynchronous copy command. No change to existing command behavior.
