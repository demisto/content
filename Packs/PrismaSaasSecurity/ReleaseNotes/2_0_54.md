
#### Integrations

##### SaaS Security Event Collector

- Fixed an issue where ingestion lag could accumulate (events arriving in XSIAM up to several hours late) when the upstream event rate was high. Each fetch execution now drains the queue until it is empty (bounded by the maximum number of iterations) instead of stopping early at the maximum number of events per fetch.
- Increased the default value of *The maximum number of iterations to retrieve events* to 150 to allow a single execution to drain a larger backlog.
- Added a high-visibility log when the event queue does not fully drain for several consecutive fetch cycles, so sustained ingestion backlog is observable.
