#### Scripts

##### ip-enrichment

- Added the **mark_mismatched_type_as_invalid** argument to control whether inputs are rejected when `extractIndicators` detects both the expected type and additional types from the same input string. Defaults to `false`, which accepts the input and uses only the expected-type indicators. This fixes an issue where valid IPs were rejected because `extractIndicators` also classified them as domains.
