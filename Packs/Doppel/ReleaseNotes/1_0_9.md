
#### Integrations

##### Doppel

- Fixed an issue in ***get-remote-data*** where the first incoming mirror sync of an incident failed because the initial **dbotMirrorLastSync** timestamp could not be parsed, leaving incidents permanently out of sync.
- Fixed an issue in ***get-modified-remote-data*** where only the first page of modified alerts was returned, so alert updates could be missed on tenants with high alert activity.
- Fixed an issue in ***get-remote-data*** where alert field updates were discarded when the alert had no audit logs. Field updates are now always mirrored, and the most recent audit-log event is attached as a War Room note only when available.
