
#### Integrations

##### Doppel

- Fixed an issue in ***fetch-incidents*** that could create many duplicate incidents from a single Doppel alert. Alerts are now de-duplicated by their Doppel alert ID both within a run and across runs.
- Updated the ***fetch-incidents*** command to process alerts page by page with a bounded number of pages per run, so a large backlog is drained safely over consecutive runs without timing out or looping.
- Updated the ***fetch-incidents*** command to fetch alerts using the maximum page size (200) supported by the Doppel API, reducing the number of requests.
- Updated the ***fetch-incidents*** command to name incidents using the external Doppel alert ID (for example, **Doppel Alert TET-1953421**) and to set the **dbotMirrorId** field explicitly for reliable mirroring and de-duplication.
- Fixed the handling of blank or invalid **Fetch Timeout** and **Maximum incidents per fetch** parameter values.
