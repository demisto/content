
#### Integrations

##### Sekoia XDR

- Fixed an issue where alerts were occasionally fetched (and mirrored) without their events. The event search window upper bound is now set to *now* instead of the alert's *last_seen_at*, so events whose timestamp is slightly later than *last_seen_at* (or indexed with a small delay) are no longer missed. Results remain scoped to the relevant alert through the `alert_short_ids` search term.

- Added automatic retries with exponential backoff on transient Sekoia API errors (HTTP 429, 500, 502, 503, 504) for all API calls, reducing failures caused by occasional *502 Bad Gateway* responses.
