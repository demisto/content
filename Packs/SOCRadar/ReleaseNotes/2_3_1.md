#### Integrations

##### SOCRadar Incidents

- Added **Critical** severity level support to incident fetching and severity mapping.
- Migrated API Key credential from `type: 4` to `type: 9` with `displaypassword` and `hiddenusername` for improved security.
- Added **Include Company ID** toggle to optionally store the configured Company ID as a custom field on each incident.

##### SOCRadar Incidents v4

- Migrated API Key credential from `type: 4` to `type: 9` with `displaypassword` and `hiddenusername` for improved security.
- Added configurable toggles (default: disabled) for enrichment fields:
  - **Include Company ID** - SOCRadar Company ID for multi-tenant identification.
  - **Include Mitigation** - Default mitigation plan from the alarm type.
  - **Include Response** - Response action steps from the alarm.
  - **Include Detection And Analysis** - Detection and analysis guidance.
  - **Include Post Incident Analysis** - Post-incident analysis recommendations.
  - **Include Compliance** - Related compliance frameworks (GDPR, ISO 27001, NIST, PCI DSS, etc.). Truncated at 3072 characters.
  - **Include Related Assets** - Related assets associated with the alarm.
  - **Include Related Entities** - Related entities (domains, CVEs, IPs, etc.).
- Added always-on incident fields: **SOCRadar Incident Link** and **SOCRadar Incident Content**.
