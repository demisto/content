## SOC Framework Package Manager 1.1.0

#### New Actions

- **`action=configure`** — Re-run integration instance, job, and lookup configuration from a pack's `xsoar_config.json` without reinstalling the pack. Use after config changes or to recover from a failed configuration step.

- **`action=sync-tags`** — Update the `value_tags` lookup dataset from the SOC Framework GitHub repository. Performs a hash-based version check and skips the upload if the data is already current. Shows the current version and hash in the war room. Use `force=true` to overwrite regardless.

#### Integration-Based Install Path

Custom pack installation now uses the **SOC Framework Pack Manager** integration instance instead of the `core-api-install-packs` command, which is unavailable on some XSIAM tenants. Packs are uploaded directly via the XSIAM content bundle endpoint using credentials stored in the integration instance params.

**Required setup:** Configure the SOC Framework Pack Manager integration instance with your XSIAM API URL, API Key ID, and API Key before running `action=apply`.

#### value_tags Versioning

`action=sync-tags` now stores a version hash in the `SOCFWTagsVersion` XSIAM List after each successful sync. Subsequent runs compare the incoming hash against the stored value and report whether an update is needed. The version is an 8-character short hash visible in the war room output.
