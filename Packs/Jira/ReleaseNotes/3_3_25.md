
#### Integrations

##### Jira Event Collector

- Added support for OAuth 2.0 authentication as an alternative to Basic authentication (username + API token).
- You can now choose between Basic and OAuth authentication methods.
- Note: OAuth authentication requires Client ID, Client Secret, Cloud ID, and Callback URL from the Atlassian Developer Console.
- Updated the integration to use the AtlassianApiModule for OAuth token management.
- Added the following new commands for OAuth 2.0 authentication management:
  - ***jira-oauth-start*** - Starts the OAuth 2.0 authentication flow and returns an authorization URL.
  - ***jira-oauth-complete*** - Completes the OAuth 2.0 authentication flow by exchanging the authorization code for tokens.
  - ***jira-oauth-test*** - Tests the OAuth 2.0 authentication by verifying the current access token is valid.
- Updated the Docker image to: *demisto/py3-tools:1.0.0.7186534*.
