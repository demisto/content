
#### Integrations

##### Demisto Lock

- Fixed an issue where ***demisto-lock-release*** could silently fail and leave the lock entry in the integration context under heavy concurrent use.
- Fixed an issue where ***demisto-lock-get*** treated unsuccessful merge attempts as success.
- Added the **Merge retry attempts** integration parameter to tune the integration-context merge retry count under contention (default 5). Each retry waits a small randomized backoff (50-200ms) to spread concurrent writers and reduce version conflicts.
