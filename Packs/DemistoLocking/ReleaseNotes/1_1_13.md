## Cortex Lock

#### Demisto Lock

- Fixed a race condition in **demisto-lock-release** (non-sync mode) where concurrent releases used a non-atomic read-modify-write pattern (`getVersionedIntegrationContext` + `setVersionedIntegrationContext`), causing "lost update" — one release would silently overwrite another, leaving locks stuck. Both sync and non-sync paths now use `mergeVersionedIntegrationContext` with retries for atomic, conflict-safe releases.
- Fixed a bug in **demisto-lock-get** (XSOAR 8 polling path) where a lock successfully written in a previous polling iteration was not recognized on re-entry because the early-exit check for `lock.guid === guid` was missing. This caused the command to poll indefinitely even after the lock was acquired.
- Improved error reporting in `attemptToAcquireLock` — write failures are now logged with a descriptive message and the function returns a boolean to signal success/failure to the caller.
