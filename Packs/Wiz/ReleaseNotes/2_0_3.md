## Wiz

- Added bidirectional incident mirroring support. Configure mirror direction, mirror limit, and comment tag to sync Wiz issue changes with XSOAR incidents.
- Added **project_ids**, **native_types**, **updated_at_before**, and **updated_at_after** filter arguments to **wiz-get-resources** command.
- Truncated issue notes to 1400 characters to prevent Wiz API errors.
- Removed deprecated **runtimeExecutionDataId** field from evidence query.
- **Breaking change:** Removed unsupported reason values from **wiz-reject-issue** (`DETECTION_EXPIRED`, `CONTROL_DISABLED`, `CONTROL_DELETED`, `OBJECT_DELETED`, `ISSUE_FIXED`) and from **wiz-resolve-issue** (`DETECTION_EXPIRED`, `CONTROL_DISABLED`, `CONTROL_DELETED`).
- Updated **wiz-reject-issue** to only offer valid rejection reasons: `FALSE_POSITIVE`, `EXCEPTION`, `WONT_FIX`.
- Updated **wiz-resolve-issue** to only offer valid resolution reasons: `OBJECT_DELETED`, `ISSUE_FIXED`, `FALSE_POSITIVE`, `EXCEPTION`, `WONT_FIX`.
- Clarified **wiz-resolve-issue** description: the command resolves Threat Detection issues only; non-Threat-Detection types (Toxic Combination, Cloud Configuration, Attack Surface) are auto-resolved when the underlying problem is fixed and should be closed via **wiz-reject-issue**.
- Improved **wiz-resolve-issue** error handling: returns a friendly `Issue not found: <id>` message when the issue is unreachable (previously crashed with `IndexError: list index out of range`), and an actionable message that suggests **wiz-reject-issue** when the issue is not a Threat Detection type.
- Fixed outgoing mirror losing the analyst's chosen close reason on incident close. The XSOAR built-in `closeReason` (Resolved / False Positive / Duplicate / Other) is now translated to the matching Wiz `resolutionReason` enum so the user-selected intent reaches Wiz instead of always defaulting to `WONT_FIX`.
- Updated the Service Account scope list in the integration setup instructions to match the scopes required by the integration's commands.

## Wiz - Outgoing Mapper

- Added new **Wiz - Outgoing Mapper** classifier (`classifier-mapper-outgoing-Wiz.json`) that carries XSOAR incident updates back to Wiz. Maps incident `status` to Wiz `status`, `dueAt` to Wiz `dueAt`, and `closeReason` / `resolutionReason` to Wiz `resolutionReason` so analyst-selected close reasons reach Wiz instead of always defaulting to `WONT_FIX`. Wired in via the integration's `defaultmapperout`; not intended for manual use.

## Wiz Mapper

- Updated the **Wiz Mapper** classifier (`classifier-Wiz_Mapper.json`) with mirror-engine incoming mappings (`dbotMirrorId`, `dbotMirrorInstance`, `dbotMirrorDirection`, `dbotMirrorTags`) sourced from the integration's mirror metadata. These let XSOAR correlate fetched Wiz Issue incidents back to the originating Wiz issue and instance for bidirectional sync.

## Wiz Defend

- Truncated issue notes to 1400 characters to prevent Wiz API errors.
- Fixed documentation for **wiz-defend-resolve-threat** to list correct resolution reason values.
- Added fallback description for detections with null descriptions from the Wiz API.
- Fixed fetch window overlap where the `before` boundary used raw current time instead of a lagged boundary.
- Fixed pagination context saving incorrect `before` value during multi-page fetches.
