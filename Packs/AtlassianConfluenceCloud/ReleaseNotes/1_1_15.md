#### Integrations

##### Atlassian Confluence Cloud

- Updated the OAuth 2.0 consent request to include the v2 Confluence Pages granular scopes (`read:page:confluence`, `write:page:confluence`, `delete:page:confluence`). This is the second half of the v1 → v2 endpoint migration introduced in 1.1.11 — without these scopes in the access token, the v2 `/wiki/api/v2/pages/{id}` endpoint that backs ***confluence-cloud-content-get*** returns **HTTP 401** even when the classic `read:confluence-content.all` scope is granted, because Atlassian treats classic and granular scopes as independent and does not honor the classic scope for v2 endpoints.
- **Action required for existing OAuth users**: After this upgrade, re-run the OAuth consent flow (`!confluence-cloud-oauth-start` → visit the URL → approve the new scopes → `!confluence-cloud-oauth-complete code=<code>`). Verify with `!confluence-cloud-oauth-test` that the returned scope list includes `read:page:confluence`. Existing access tokens minted before this upgrade do not carry the new scopes; only a fresh consent will produce a token that succeeds against ***confluence-cloud-content-get***.
