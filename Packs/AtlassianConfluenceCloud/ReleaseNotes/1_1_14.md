#### Integrations

##### Atlassian Confluence Cloud

- Fixed an issue where the Base URL would not be constructed correctly when using **OAuth 2.0** authentication. OAuth 2.0 requests now correctly go through Atlassian's OAuth 2.0 (3LO) gateway at `https://api.atlassian.com/ex/confluence/{cloudId}` instead of the site URL, which resolves the **HTTP 403** *"Current user not permitted to use Confluence"* error returned by `fetch-events` and other commands.
- Updated the Docker image to: *demisto/python3:3.12.13.9059085*.
