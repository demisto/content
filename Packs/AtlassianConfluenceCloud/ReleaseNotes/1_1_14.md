#### Integrations

##### Atlassian Confluence Cloud

- Fixed an issue where the Base URL would not be constructed correctly.
- Fixed an issue where the ***confluence-cloud-content-get*** command returned **HTTP 401 Unauthorized** when used with **OAuth 2.0** authentication. The command now targets the Confluence Cloud v2 Pages API (`GET /wiki/api/v2/pages/{id}`), which is exposed via Atlassian's OAuth 2.0 (3LO) gateway, instead of the legacy v1 `/wiki/rest/api/content/{id}` endpoint, which is not. The v2 endpoint also works with Basic Authentication, so no behavior change is expected for existing Basic Auth users.
- Under OAuth 2.0, the ***confluence-cloud-content-get*** command now requires the `read:page:confluence` granular scope. Update your Atlassian OAuth 2.0 app permissions and re-run the OAuth consent flow if you previously hit a 401 on this command.
- Note: The output format of ***confluence-cloud-content-get*** has changed to match the v2 `PageSingle` schema. The most notable differences from the previous v1 shape are `spaceId` (instead of nested `space.name`), `authorId`/`ownerId` (instead of `history.createdBy`), and `body.storage.value` is now the top-level body field (instead of `body.storage.value` nested inside the v1 envelope).
