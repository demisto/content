
#### Integrations

##### BMC Helix ITSM

- Added support for **Client ID** parameter that required when 'use oauth' is selected.
- Added support for **Authorization Code** parameter that obtained by running the **bmc-itsm-generate-login-url** command and following the url. required when 'use oauth' is selected.
- Added support for **Redirect URI** parameter that required when 'use oauth' is selected.
- Added support for **OpenID Issuer URL** parameter that the rsso server url. for example https://<rsso_server_name>:8443/rsso. required when 'use oauth' is selected.
- Added support for **Use OAuth** parameter that select this option to use oauth 2.0 authorization code flow for authentication instead of username/password.
- Added support for **bmc-itsm-add-attachment** command that adds attachments to an existing bmc helix itsm ticket. supports adding attachments to incident, change request, service request, task, problem investigation, known error, and work order forms.
- Added support for **bmc-itsm-generate-login-url** command that generate the login url used for the oauth 2.0 authorization code flow. follow the url to obtain an authorization code, then paste it into the instance configuration *authorization code* field.
