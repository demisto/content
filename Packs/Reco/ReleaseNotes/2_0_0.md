#### Integrations

##### Reco
- **Breaking change**: The ***fetch-incidents*** *Risk level* parameter now fetches alerts at the given severity and above, instead of an exact match. For example, setting it to *MEDIUM* now also returns *HIGH* and *CRITICAL* alerts.
- **Breaking change**: On ***reco-get-risky-users***, *Reco.RiskyUsers.email_account* was renamed to *Reco.RiskyUsers.accountEmail*.
- **Breaking change**: On ***reco-get-user-context-by-email-address***, *Reco.User.email_account* was renamed to *Reco.User.email*, and *Reco.User.full_name* was renamed to *Reco.User.name*. The *Reco.User.groups*, *Reco.User.labels*, and *Reco.User.category* fields were removed, with no replacement.
- **Breaking change**: On ***reco-get-sensitive-assets-by-id***, ***reco-get-sensitive-assets-by-name***, and ***reco-get-assets-by-id***, several *Reco.SensitiveAssets* fields were renamed: *file_name* to *name*, *file_owner* to *owner*, *file_url* to *url*, *sensitivity_level* to *sensitivityLevel*, and *visibility* to *permissionVisibility*. The *currently_permitted_users* and *source* fields were removed, with no replacement.
- **Breaking change**: On ***reco-get-apps***, *Reco.Apps.app_id* was renamed to *Reco.Apps.id*, *Reco.Apps.app_name* was renamed to *Reco.Apps.name*, and *Reco.Apps.data_access* was renamed to *Reco.Apps.authorization* (now the app's authorization/sanction status rather than its data access level). The *Reco.Apps.risk_score*, *Reco.Apps.status*, and *Reco.Apps.created_at* fields were removed, with no replacement.
- **Breaking change**: On ***reco-set-app-authorization-status***, *Reco.AppAuthorization.rows_affected* is no longer populated, since the External API's update response carries no row count.
- Updated the alert fetching, identity, app, file, and label commands to use the new Reco External API for cleaner JSON responses and improved reliability.
- Added support for rate-limit handling (automatic 429 retry with exponential backoff) for all External API calls.
- Added the following commands, all backed by the External API and accepting SCIM v2 filter expressions: ***reco-list-events***, ***reco-list-posture-issues***, ***reco-list-accounts***, ***reco-list-devices***, ***reco-list-ai-agents***, ***reco-list-groups***, ***reco-list-saas-to-saas***, ***reco-list-ip-addresses***, ***reco-list-business-units***, ***reco-list-audit-logs***, ***reco-list-posture-checks***, ***reco-list-threat-detection-policies***, ***reco-list-exclusions***, and ***reco-list-app-instances***.
