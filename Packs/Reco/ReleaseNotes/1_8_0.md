#### Integrations

##### Reco
- Migrated alert fetching, identity, app, file, and label commands to the new Reco External API (`/api/v1/external-api/`) for cleaner JSON responses and improved reliability.
- Added support for multiple severity filters in `fetch-incidents` — the **Risk level** parameter now accepts a comma-separated list (e.g., `HIGH,CRITICAL`). Resolves PRO-303.
- Added rate-limit handling (automatic 429 retry with exponential backoff) for all External API calls.
- Added five new commands: **reco-list-events**, **reco-list-posture-issues**, **reco-list-accounts**, **reco-list-devices**, **reco-list-ai-agents**. All accept SCIM v2 filter expressions.
- Updated context output field names for **reco-get-risky-users**, **reco-get-user-context-by-email-address**, **reco-get-sensitive-assets-by-id**, **reco-get-assets-by-id**, and **reco-get-apps** to match the External API response schema.
