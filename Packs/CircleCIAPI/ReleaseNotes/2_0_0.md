## CircleCI API

Initial release of the CircleCI API content pack by GoCortexIO.

The pack integrates the CircleCI v2 API with Cortex, collecting CircleCI control-plane and
pipeline activity into Cortex datasets, mapping it to the XDM data model, and detecting CI/CD
security risks with correlation rules. Detection content is grounded in the OWASP CI/CD Top 10,
with a focus on Poisoned Pipeline Execution, persistence, and secrets exposure.

#### Integrations

- **CircleCI Pipelines Event Collector** collects pipeline activity from `GET /pipeline` into the
  **circleci_pipelines_raw** dataset, using rolling high-water-mark collection so a delayed or
  overlapping poll never leaves a gap.
- **CircleCI Webhooks Event Collector** collects the outbound webhook inventory into the
  **circleci_webhooks_raw** dataset as periodic snapshots.
- **CircleCI Triggers Event Collector** collects the pipeline trigger inventory (scheduled
  triggers with cron expression and attribution actor, plus push triggers) into the
  **circleci_triggers_raw** dataset.
- **CircleCI Contexts Event Collector** collects the shared-context and context
  environment-variable name inventory into the **circleci_context_envvars_raw** dataset.
  Environment-variable values are masked by the API; only names are collected.
- **CircleCI Project Settings Event Collector** collects the advanced project-settings posture,
  including the Poisoned Pipeline Execution preconditions, into the
  **circleci_project_settings_raw** dataset.
- **CircleCI Checkout Keys Event Collector** collects the deploy/checkout key inventory into the
  **circleci_checkout_keys_raw** dataset. This collector is under development and disabled by
  default; the v2 API exposes only deploy/checkout keys, not the separate Additional SSH Keys
  surface.

Collectors auto-discover projects from recent pipeline activity where possible, so project IDs do
not need to be tracked manually.

#### Modeling Rules

- Added the **CircleCI API Modeling Rule**, which maps every populated dataset to the XDM data
  model, including actor identity, organisation and project, resource details, and a per-record
  risk classification for the inventory datasets.

#### Correlation Rules

- **CircleCI - New Pipeline Created** alerts on a newly created pipeline (MITRE TA0002 / T1072).
- **CircleCI - New Outbound Webhook Created** alerts on a newly created outbound webhook, a
  potential exfiltration channel (MITRE TA0010 / T1567).
- **CircleCI - New Pipeline Trigger Created** alerts on a newly created scheduled or push
  trigger, a persistence vector (MITRE TA0003 / T1053).
- **CircleCI - New Context or Context Secret** alerts on a newly created shared context or secret
  name, the data class exfiltrated in the January 2023 CircleCI incident (MITRE TA0006 / T1552).
- **CircleCI - Insecure Fork Build Settings** alerts on a project where forked pull requests can
  build or receive the project's secrets, a Poisoned Pipeline Execution precondition
  (MITRE TA0001 / T1195).
