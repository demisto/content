#### Scripts

##### SendEmailReply

- Fixed an issue where email replies were not grouped into the same conversation thread in the recipient's email client. The *References* header now includes the full chain of all prior Message-IDs from the thread context (RFC 2822 compliant), ensuring proper threading in Gmail, Outlook, and other email clients.
- Fixed an issue where the outgoing reply subject included an internal XSOAR tracking code (e.g., `<code> original subject`), which caused Gmail to treat the first reply as a new conversation instead of grouping it with the original thread. The outgoing subject is now sent as the original email subject.

##### PreprocessEmail

- Added a fallback routing mechanism for incoming reply emails that do not contain the XSOAR tracking code in the subject. The script now attempts to match the reply to an existing incident by comparing the email's *In-Reply-To* message ID against the *emaillatestmessage* field of open incidents, ensuring replies are correctly routed even when the outgoing subject does not include the tracking code.
