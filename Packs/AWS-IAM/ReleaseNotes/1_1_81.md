
#### Integrations

##### AWS - Identity and Access Management

- Added support for ***Deactivate User IAM Access Key* - aws-iam-access-key-update-quick-action** command that instantly disables a specific iam access key. this stops all cli, sdk, and apibased communication for this key id without deleting it, allowing for forensic recovery or reactive "reenabling" if the key is found to be safe.
- Added support for ***Block All User Actions* - aws-iam-user-policy-put-quick-action** command that attaches an inline "deny all" policy for the specified iam user, effectively blocking the user from performing any actions on all aws resources.
- Added support for ***Detach IAM Role from EC2 Instance* - aws-iam-role-from-instance-profile-remove-quick-action** command that detaches the iam role from the specified instance profile. this immediately prevents the ec2 instance from using any assigned aws permissions, effectively "quarantining" its identity without stopping the server.
- Added support for ***Remove User Console Access* - aws-iam-login-profile-delete-quick-action** command that permanently deletes the iam user's password (login profile). this prevents the user from signing into the aws management console but does not deactivate their programmatic access keys or delete the iam user itself.
