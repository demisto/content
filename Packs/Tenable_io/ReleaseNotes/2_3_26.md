
#### Integrations

##### Tenable Vulnerability Management (formerly Tenable.io)

- Fixed an issue where vulnerabilities were not being ingested quickly enough and were causing incomplete data in the system.
- Added a new advanced configuration parameter, **Max vulnerability chunks per fetch**, to control the maximum number of vulnerability chunks downloaded in a single fetch (default ``8``, capped at ``50``).
- Increased the XSIAM payload chunk size from 1 MiB to 4.5 MB for both assets and vulnerabilities, reducing the number of sequential ingest API calls per fetch cycle.
- Parallelized the XSIAM POST loop for vulnerabilities (single-threaded path is preserved for assets to maintain snapshot sealing semantics). The integration explicitly waits on all parallel POSTs to complete before continuing, so any failed POST is surfaced and the lastrun cleanup is correctly skipped on error.
- Combined effect: a fetch cycle that previously took ~17 minutes for 25 vulnerability chunks is expected to drop to ~2-3 minutes, a ~7-8x throughput improvement, with no change to ordering or snapshot-sealing semantics.
