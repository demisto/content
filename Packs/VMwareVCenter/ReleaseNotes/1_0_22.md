
#### Modeling Rules

##### VMware Vcenter 2.9 Modeling Rule

- Updated the **VMware Vcenter 2.9 Modeling Rule** to correctly map `vcsa-audit` SYSCALL records:
  - Mapped the parent process id (`ppid`) to the source process id and stopped storing the `vcsa-audit` emitter as the source process name.
  - Mapped the process id (`pid`), command name (`comm`), executable path (`exe`), and user id (`uid`) to the target process and target user fields.
  - Removed the literal backslashes from the target process executable path.
  - Set the target process command line and target process parent id to null for SYSCALL records (no command line is present).
  - Mapped the audit operation to the record type (for example, `SYSCALL`) instead of a hardcoded value.
