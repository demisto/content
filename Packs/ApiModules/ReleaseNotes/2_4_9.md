#### Scripts

##### AtlassianApiModule

- Added the v2 Confluence Pages API granular OAuth scopes (`read:page:confluence`, `write:page:confluence`, `delete:page:confluence`) to the Confluence Cloud OAuth 2.0 (3LO) consent request. These scopes are required by the Confluence v2 Pages API (`/wiki/api/v2/pages/{id}`). Without them, Atlassian's OAuth gateway returns **HTTP 401** when callers hit v2 page endpoints, even when the classic `read:confluence-content.all` / `write:confluence-content` scopes are granted, because those classic scopes do not cover the v2 endpoints.
- **Action required for existing OAuth users**: After upgrading, re-run the OAuth consent flow for any Confluence Cloud integration instance (`!confluence-cloud-oauth-start` → visit URL → `!confluence-cloud-oauth-complete code=…`) so the access token is re-issued with the new scope set. Tokens minted before the upgrade do not gain the new scopes automatically.
