
#### Scripts

##### CoreIRApiModule

- Improved error handling for the endpoint scan and abort-scan commands. Invalid *endpoint_id_list* values (for example, a hostname or a truncated ID) are now rejected with a clear message before the request is sent, and failures to create a scan action now return a specific reason (such as no matching endpoints, an active scan, a non-scannable status, or an unsupported agent version) instead of a generic server-side error.
