#### Integrations

##### Stairwell

- **Breaking Changes**:
  - **Removed YARA Management Commands**: Removed `stairwell-yara-list-rules`, `stairwell-yara-update-rule`, and `stairwell-yara-delete-rule` commands. These commands are now managed through the Stairwell web UI. The following YARA commands remain available:
    - `stairwell-yara-create-rule` - Create new YARA rules
    - `stairwell-yara-get-rule` - Retrieve specific YARA rule details
    - `stairwell-yara-query-matches` - Query objects matching a YARA rule

- **Enhanced File Intake Upload** (`stairwell-intake-upload`):
  - **Multiple File Source Methods**: Added support for three flexible file source methods:
    - **entryID** - Upload files from XSOAR War Room context (recommended for production playbooks)
    - **url** - Download and upload files from HTTP/HTTPS URLs with streaming support
    - **filePath** - Direct file path access (existing behavior, maintains backward compatibility)
  - **Intelligent File Resolution**: Automatically resolves file source and validates exactly one source is provided
  - **Streaming Downloads**: Large files (>100MB) are downloaded in 1MB chunks to prevent memory exhaustion
  - **Automatic Cleanup**: Temporary files from URL downloads are automatically cleaned up using finally blocks
  - **Enhanced Error Handling**: Clear error messages for file source validation, URL scheme validation, and network errors
  - **Updated Parameters**:
    - Changed `filePath` from required to optional (one of entryID, url, or filePath must be provided)
    - Added `entryID` parameter for War Room file integration
    - Added `url` parameter for HTTP/HTTPS file downloads
    - `sha256` parameter remains optional with automatic calculation

- **Bug Fixes**:
  - **YARA Query Matches**: Fixed `stairwell-yara-query-matches` command to correctly parse API response. The command was looking for `matchingObjects` but the API returns `objects`, causing "No matches found" to display even when matches existed.

- **Documentation Enhancements**:
  - **Multi-line YARA Rules**: Added important documentation about using backticks (`) instead of quotes for multi-line YARA rule definitions in command_examples and integration description
  - **Comprehensive Integration Description**: Completely rewrote `Inception_description.md` with:
    - Detailed command reference for all 33+ commands
    - Three file source methods with examples
    - Common workflow patterns (File Analysis, File Upload & Analysis, Threat Hunting)
    - Network intelligence commands documentation
    - Asset management commands documentation
    - Integration features (reliability, security, error handling)
    - Configuration guidance
  - **Updated README**: Enhanced pack-level and integration-level README files with current feature set

- **Code Quality**:
  - Added helper function `_resolve_file_source()` for clean separation of file resolution logic
  - Implemented proper temp file tracking with guaranteed cleanup
  - Enhanced import statements (added `urllib.parse` and `tempfile`)
  - Improved error messages for better troubleshooting

- **Backward Compatibility**:
  - All existing playbooks using `filePath` continue to work without modification
  - SHA256 auto-calculation behavior unchanged
  - All output context paths remain consistent
