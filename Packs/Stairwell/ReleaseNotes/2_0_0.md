## Stairwell Integration v2.0.0

### Breaking Changes
- **Integration ID Changed**: `Inception` → `Stairwell`
- **Command Names Changed**: All commands now use `stairwell-*` prefix instead of `inception-*`
  - `inception-variant-discovery` → `stairwell-variant-discovery`
  - `inception-file-enrichment` → `stairwell-file-enrichment`
  - `inception-metadata-get` → `stairwell-metadata-get`
  - And all other commands have been renamed with the new prefix
- **Context Paths Changed**: `Inception.*` → `Stairwell.*`
- **Display Name Changed**: "Stairwell Inception" → "Stairwell"

⚠️ **Migration Required**: Existing playbooks and scripts using the Inception integration must be updated to use the new command names and context paths.

### New Features
- **Network Intelligence**: New commands for network-based threat intelligence
- **YARA Rules**: Search and management of YARA rules
- **Asset Management**: Commands for managing assets and asset groups
- **Intake System**: File upload and processing with detonation support
  - Upload files from War Room, URLs, or file paths
  - Automatic file analysis and processing
  - Detonation plan support
  - Origin tracking (web referrers, zones)
- **Object Operations**: Sightings tracking, detonation triggers, opinions management
  - `stairwell-object-sightings` - List sightings for an object
  - `stairwell-object-detonation-trigger` - Trigger detonation
  - `stairwell-object-detonation-get` - Get detonation results
  - `stairwell-object-opinion-list` - List opinions for an object
- **Enhanced Metadata**: Comprehensive metadata extraction and analysis

### Improvements
- **Enhanced Error Handling**: Significantly improved error messages across all commands
  - Added timeout exception handling with clear, actionable messages
  - Protected type conversion operations to prevent crashes from invalid input
  - Added missing exception handling to object-related commands (sightings, detonation, opinions)
  - Improved error messages with context and troubleshooting guidance
- **Better HTTP Error Reporting**: Standardized HTTP status code error messages
  - 401: Clear message about API key authentication
  - 404: Specific message about missing resources
  - 429: Rate limit guidance
  - 500-504: Server/gateway error guidance
- **Improved Debugging**: Enhanced logging for file cleanup operations
- **Input Validation**: Better validation for numeric parameters (pageSize, originZoneId)

### Bug Fixes
- Fixed potential crashes from timeout exceptions in file upload and download operations
- Fixed potential crashes from invalid page_size parameters in YARA and Asset commands
- Fixed potential crashes from invalid originZoneId parameter in intake upload
- Fixed missing error handling in object sightings, detonation trigger, detonation get, and opinions commands
- Fixed JSON parsing errors in intake preflight response handling
