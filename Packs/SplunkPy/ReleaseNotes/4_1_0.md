#### Integrations

##### SplunkPy v2

- Added the **Maximum investigations per fetch** parameter (max investigations per cycle, hard-clamped to 100).
- Added the **Event types to fetch** parameter to select which Splunk event types (Finding, Investigation) to ingest. Each event is tagged with `splunk_es_event_type` for classifier routing.
- Added the **First fetch timestamp (Investigations)** parameter (e.g. `12 hours`, `7 days`, `3 months`).
- Added the **Investigations fetch query** parameter for the investigations SPL. The literal `custom_filter_placeholder` token is replaced at runtime with the time-window and pagination arguments.
- Updated the **splunk-finding-event-edit** command to no longer use the *finding_time* argument.
