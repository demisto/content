
#### Integrations

##### KOI

- Added support for 13 new commands:
  - ***koi-devices-list*** - Lists devices registered with KOI.
  - ***koi-device-inventory-get*** - Lists the items installed on a specific device.
  - ***koi-findings-list*** - Lists finding (detection) definitions.
  - ***koi-groups-list*** - Lists device groups.
  - ***koi-users-list*** - Lists users.
  - ***koi-remediations-list*** - Lists remediation suggestions.
  - ***koi-approval-requests-list*** - Lists approval requests.
  - ***koi-runtime-policies-list*** - Lists agent runtime (hardening) policies.
  - ***koi-runtime-policy-get*** - Retrieves a single runtime (hardening) policy by ID.
  - ***koi-koidex-search*** - Searches the KOI catalog for items by marketplace and term.
  - ***koi-koidex-risk-report*** - Retrieves the risk and compliance report for a KOI catalog item.
  - ***koi-fetch-context-get*** - Displays the integration's event-collection fetch state for diagnostics.
  - ***koi-fetch-context-set*** - Modifies the integration's event-collection fetch state for maintenance.
- Added the *First Fetch Time Range* parameter to control the initial event-collection lookback window.

#### Modeling Rules

##### Koi Modeling Rule

- Added the **Koi Modeling Rule** to map KOI alert and audit events to the Cortex Data Model (XDM).

#### Parsing Rules

##### Koi Parsing Rule

- Added the **Koi Parsing Rule** to normalize raw KOI events and promote alert and audit fields for modeling.
