## Koi Event Collector

This integration fetches alerts and audit logs from the Koi API for ingestion into Cortex XSIAM.

### Authentication

The Koi API uses API key-based authentication via a Bearer token.

#### Creating an API Key

1. **Ensure Appropriate Role**: You must have the `xt-Administrator` role to create an API key.
2. **Navigate to Settings**: Access the Settings page from the top navigation bar.
3. **Open API Access Tab**: In the Settings page, select the **API Access** tab.
4. **Create New API Key**: Click the **Create new API key** button.
5. **Copy Your Key**: Within a few seconds, a new API key will appear in the table. Click the **Copy** button next to the key to copy it securely.

> **Note**: API keys can be set to expire up to 1 year after creation. Notification emails are sent 30, 7, and 3 days before expiration.

### Rate Limiting

The Koi API enforces a rate limit of **30 requests per minute per route**. The integration is designed to respect these limits.

### Event Types

- **Alerts**: Security alerts generated by the Koi platform.
- **Audit Logs**: Administrative and operational audit events. Can be filtered by type:
  - `approval_requests`, `devices`, `endpoints`, `extensions`, `firewall`, `guardrails`, `notifications`, `policies`, `remediation`, `requests`, `settings`, `vetting`
