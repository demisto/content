#### Integrations

##### Cortex Platform Core

- Updated the **core-run-playbook** command to accept a playbook name or ID via the new `playbook` argument. The name is automatically resolved to an ID using the playbooks metadata API. If the name is not unique, an error is raised with the conflicting IDs so the user can provide the ID directly instead. The `playbook_id` argument is deprecated in favor of `playbook`.
