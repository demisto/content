#### Integrations

##### Cortex Platform - Core

- Updated the ***core-update-case*** command's *custom_fields* argument to support a new preferred dict format: `{"field1": "value1", "multiselect_field": ["a", "b"]}`. The legacy list-of-objects format `[{"field1": "value1"}, {"field2": "value2"}]` remains supported for backward compatibility.
- Fixed an issue in the ***core-update-case*** command where updating a multi-select custom field with a string value instead of a list caused an unhandled 500 error.
