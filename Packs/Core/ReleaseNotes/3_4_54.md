
##### Cortex Platform - Core

- Enhanced ***core-get-cases*** command to support a variety of filtering options and updated some context outputs:
  - Added new arguments to filter cases:
    - **case_domain**
    - **severity**
    - **asset_ids**
    - **asset_groups**
    - **hosts**
    - **assignee**
    - **case_name**
    - **case_description**
    - **tag**

  - Added new argument to retrieve enriched case data:
    - **get-enriched-case-data**

  - Expanded outputs:
    - Added new outputs:
      - **Core.Case.asset_accounts**
      - **Core.Case.asset_group_ids**
      - **Core.Case.asset_ids**
      - **Core.Case.asset_names**
      - **Core.Case.asset_providers**
      - **Core.Case.asset_regions**
      - **Core.Case.asset_types**
      - **Core.Case.CaseExtraData.issue_ids**
      - **Core.Case.CaseExtraData.file_artifacts**
      - **Core.Case.CaseExtraData.network_artifacts**
      - **Core.Case.CaseExtraData.starred_manually**
      - **Core.Case.CaseExtraData.xdr_url**
      - **Core.Case.CaseExtraData.manual_description**
      - **Core.Case.CaseExtraData.notes**
      - **Core.Case.CaseExtraData.detection_time**
      - **Core.CasesMetadata.returned_count**
      - **Core.CasesMetadata.filtered_count**

  - The following outputs have been removed and are now retrievable using **get-enriched-case-data** argument set to true:
    - **Core.Case.detection_time**
    - **Core.Case.notes**
    - **Core.Case.xdr_url**
    - **Core.Case.manual_description**
    - **Core.Case.starred_manually**
