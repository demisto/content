
#### Integrations

##### Jamf v2

- Updated the ***jamf-computer-lock*** command to use the Jamf Pro MDM API. Added the *phone_number* and *management_id* arguments.
- Updated the ***jamf-computer-erase*** command to use the dedicated computer erase endpoint. The command is no longer deprecated.
- Updated the ***jamf-mobile-device-lost-mode*** command to use the Jamf Pro MDM API. Added the *lost_mode_phone*, *lost_mode_footnote*, and *management_id* arguments. The command is no longer in beta. **Breaking Change**: The context output was updated - the *status* output was removed and the *href* output was added.
- Updated the ***jamf-mobile-device-erase*** command to use the dedicated mobile device erase endpoint. Added the *disallow_proximity_setup*, *clear_activation_lock*, and *return_to_service* arguments. Deprecated the *clear_activation_code* argument (now an alias of *clear_activation_lock*). The command is no longer in beta. **Breaking Change**: The context output was updated.
- Updated the ***jamf-get-users*** command to use the Jamf Pro API with server-side pagination.
- Added the ***jamf-mdm-command-status*** command to retrieve the status of MDM commands.
