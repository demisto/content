#### Integrations

##### PANOS-HA

- NEW: Added device_priority parameter support for HA election control
- Enhanced: Device priority configuration (0-255, default: 100) determines which firewall becomes active
- Enhanced: Higher priority device will become active in HA pair
- Enhanced: Debug logging now shows device_priority value during configuration
- Updated: Parameter validation includes device priority in configuration display

#### Feature Details

**Device Priority** controls which firewall in an HA pair becomes the active device during election.

**How It Works:**
- Range: 0-255
- Default: 100
- Higher number = Preferred active device
- Example: Device with priority 150 will become active over device with priority 100

**Use Cases:**
1. **Preferred Active Device**: Set higher priority (e.g., 150) on the firewall you want to be active
2. **Equal Priority**: Leave both at default (100) for device-agnostic HA
3. **Forced Passive**: Set lower priority (e.g., 50) on a device you want to remain passive

**Example:**

```bash
# PA2 configured with priority 150 (preferred active)
!panos-ha-configure using="pa2" group_id="10" device_priority="150" peer_ip="192.168.26.101" ...

# PA1 configured with default priority 100
!panos-ha-configure using="pa1" group_id="10" peer_ip="192.168.26.102" ...

# Result: PA2 will become active due to higher priority (150 > 100)
```

- Parameter is optional
- Default value: 100 (PAN-OS standard)
- Existing configurations without device_priority will use default value
