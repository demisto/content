#### Integrations

##### PANOS-HA

- NEW: Full support for HA backup links configuration
- NEW: Added peer_ip_backup parameter support for backup HA1 control link peer addressing
- NEW: Added ha1_backup_port, ha1_backup_ip_address, ha1_backup_netmask parameters for HA1 backup control link
- NEW: Added ha2_backup_port, ha2_backup_ip_address, ha2_backup_netmask parameters for HA2 backup data link
- Improved: Interface validation now includes HA2 backup port verification
- Enhanced: panos-ha-configure command now supports complete HA configuration with redundant links

#### Technical Details

This release adds comprehensive backup link support for PAN-OS High Availability configuration.

**New Capabilities:**

1. **Peer IP Backup**: Configure backup peer IP address for HA1 redundancy
   - Parameter: `peer_ip_backup`
   - Used when primary HA1 link fails

2. **HA1 Backup Link**: Configure backup control link for control plane redundancy
   - Parameters: `ha1_backup_port`, `ha1_backup_ip_address`, `ha1_backup_netmask`
   - Typical ports: ha1-b, ethernet interface
   - Provides redundancy for heartbeat and configuration sync

3. **HA2 Backup Link**: Configure backup data link for session synchronization redundancy
   - Parameters: `ha2_backup_port`, `ha2_backup_ip_address`, `ha2_backup_netmask`
   - Typical ports: ethernet interfaces
   - Provides redundancy for session table synchronization

**Example Usage:**

```bash
!panos-ha-configure using="pa3" \
  group_id="10" \
  peer_ip="192.168.26.101" \
  peer_ip_backup="192.168.26.105" \
  passive_link_state="auto" \
  ha1_port="ha1-a" \
  ha1_ip_address="192.168.26.102" \
  ha1_netmask="255.255.255.252" \
  ha1_backup_port="ha1-b" \
  ha1_backup_ip_address="192.168.26.106" \
  ha1_backup_netmask="255.255.255.252" \
  ha2_port="ethernet1/11" \
  ha2_ip_address="192.168.26.110" \
  ha2_netmask="255.255.255.252" \
  ha2_backup_port="ethernet1/12" \
  ha2_backup_ip_address="192.168.26.114" \
  ha2_backup_netmask="255.255.255.252"
```

**Backward Compatibility:**
- All new parameters are optional
- Existing configurations without backup links continue to work
- Minimum required parameters remain unchanged (group_id, peer_ip, ha1/ha2 primary links)
