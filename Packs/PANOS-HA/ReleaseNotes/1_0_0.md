#### Integrations

##### PANOS-HA

- **NEW:** Initial release of PAN-OS High Availability integration.
- **FEATURE:** Interface validation before HA configuration to prevent errors.
- **FEATURE:** Added `panos-ha-list-interfaces` command to list all available network interfaces.
- **FEATURE:** Added `panos-ha-validate-interfaces` command to validate interface existence before configuration.
- **ENHANCEMENT:** Updated `panos-ha-configure` command with automatic interface validation.
- **CRITICAL:** HA interface configuration now validates that specified interfaces exist on the device before attempting to apply configuration, preventing configuration failures.

#### Commands

- panos-ha-get-state: Retrieves current HA state of a firewall
- panos-ha-get-config: Retrieves HA configuration from a firewall
- panos-ha-suspend-peer: Suspends local firewall forcing failover
- panos-ha-make-peer-functional: Brings suspended/passive firewall to functional state
- panos-ha-sync-config: Synchronizes running configuration to peer
- panos-ha-sync-state: Synchronizes session state to peer
- panos-ha-configure: Configures HA setup with automatic interface validation
- panos-ha-enable: Enables HA functionality
- panos-ha-disable: Disables HA functionality
- panos-ha-list-interfaces: Lists all available network interfaces
- panos-ha-validate-interfaces: Validates specified interfaces exist
- panos-panorama-ha-reconfigure: Issues HA reconfiguration command to Panorama

#### Use Cases

- Automated failover orchestration for perimeter firewalls
- HA configuration management with validation
- Disaster recovery automation
- Firewall maintenance workflows
