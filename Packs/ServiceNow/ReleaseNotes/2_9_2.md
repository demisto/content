
#### Integrations

##### ServiceNow v2

- Fixed an issue where the ***servicenow-create-item-order*** command failed with a `400` error when the integration was configured with API version **v2**. The `servicecatalog/items/{id}/order_now` endpoint only supports API v1, so the URL is now automatically downgraded to v1 for this endpoint.
- Added the optional **no_validation** argument to the ***servicenow-create-item-order*** command. When set to **true**, the `sysparm_no_validation` flag is included in the request body, allowing catalog item orders to bypass ServiceNow form validation. This is useful when catalog items have required attachments or custom validation logic that cannot be satisfied at order-creation time. The attachment can be uploaded separately using the ***servicenow-upload-file*** command.
