
#### Integrations

##### AWS - GuardDuty Event Collector

- Fixed an issue where critical findings could be skipped when a fetch cycle was truncated by the configured event limit. The fetch cursor is no longer advanced into a partially-consumed ``UpdatedAt`` second, guaranteeing that same-second findings are not lost between cycles.
- Added the **Exclude archived/suppressed findings** parameter. When enabled, findings archived in GuardDuty (including those archived by suppression rules) are no longer fetched.
