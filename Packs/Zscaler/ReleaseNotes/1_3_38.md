
#### Integrations

##### Zscaler Internet Access

- Added the *category-name* argument to the following commands to support Zscaler API changes that require the category name for custom category operations:
  - ***zscaler-category-add-url***
  - ***zscaler-category-add-ip***
  - ***zscaler-category-remove-url***
  - ***zscaler-category-remove-ip***
- Fixed an issue where modifying custom URL categories (e.g., CUSTOM_01) would fail with HTTP 400 errors due to missing `configuredName` and `superCategory` fields required by the Zscaler API for custom category operations.
