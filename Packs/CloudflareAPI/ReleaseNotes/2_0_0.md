## Cloudflare API

Initial release of the Cloudflare API content pack by GoCortexIO.

The pack integrates the Cloudflare API with Cortex, collecting Cloudflare account, security, and
Zero Trust Access activity into Cortex datasets, mapping it to the XDM data model, and detecting
account-takeover and configuration-abuse scenarios with correlation rules.

#### Integrations

- **Cloudflare Audit Logs Event Collector** collects account and configuration activity from
  `GET /accounts/{id}/audit_logs` into the **cloudflare_account_audit_raw** dataset, with
  per-account rolling high-water-mark collection.
- **Cloudflare Security Insights Event Collector** collects Security Center findings into the
  **cloudflare_security_insights_raw** dataset as periodic snapshots.
- **Cloudflare Access Authentication Logs Event Collector** collects Zero Trust Access
  authentication logs into the **cloudflare_access_auth_raw** dataset. This endpoint returns
  data only for accounts that use Cloudflare Zero Trust Access.

#### Modeling Rules

- Added the **Cloudflare API Modeling Rule**, which maps the account audit and security insights
  datasets to the XDM data model. Audit events are classified by operation verb (create, update,
  delete, read, audit) with dashboard logins mapped as authentication events; security findings
  are mapped to the XDM alert fields.

#### Correlation Rules

- **Cloudflare - API Token Created or Rolled** alerts on a new or rolled API token, a
  persistence vector after account compromise (MITRE TA0003 / T1098).
- **Cloudflare - Bulk DNS Record Deletion** alerts when an actor deletes many DNS records in a
  short window, a destructive or disruptive action (MITRE TA0040 / T1565).
- **Cloudflare - Security or Configuration Setting Changed** alerts on a change to a security or
  firewall setting (MITRE TA0005 / T1562).
- **Cloudflare - Security Posture Finding** alerts on a Security Center finding above an
  informational severity.
