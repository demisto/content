Workday IAM Integration
## Configure Workday IT Admin on Cortex XSOAR

1. Navigate to **Settings** > **Integrations** > **Servers & Services**.
2. Search for Workday IT Admin.
3. Click **Add instance** to create and configure a new integration instance.

| **Parameter** | **Description** | **Required** |
| --- | --- | --- |
| credentials | Username | False |
| report_type | Report Type | False |
| report_url | Workday Report URL | True |
| fetch_events_time_minutes | Fetch Events Frequency In minutes | False |
| fetch_limit | Fetch Limit \(Recommended less than 200\) | False |
| email_notification_ids | Email Notification Ids \(Separated by comma\) | False |
| smtp_server | SMTP Server Host | False |
| smtp_port | SMTP Server Port | False |
| from_email | From Email | False |
| isFetch | Fetch incidents | False |
| incidentType | Incident type | False |
| insecure | Trust any certificate \(not secure\) | False |
| proxy | Use system proxy settings | False |

4. Click **Test** to validate the URLs, token, and connection.

### workday-get-delta-user-report
***
Gets the events based on from and to date. Based on how the report is filtered on Workday, it may be completion date or effective date


#### Base Command

`workday-get-delta-user-report`
#### Input

| **Argument Name** | **Description** | **Required** |
| --- | --- | --- |
| from_date | Report start date time in format "%Y-%m-%dT%H:%M:%SZ" Ex: 2020-01-20T11:48:34.336-08:00 | Required | 
| to_date | Report end date time in format "%Y-%m-%dT%H:%M:%SZ". Ex: 2020-01-20T11:48:34.336-08:00 | Required | 


#### Context Output

There is no context output for this command.

#### Command Example
```!workday-get-delta-user-report using="Workday IT Admin - Delta Report" from_date=2020-08-11T08:00:00.000-07:00 to_date=2020-08-12T08:00:00.000-07:00```
