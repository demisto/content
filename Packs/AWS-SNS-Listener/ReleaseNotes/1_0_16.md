
#### Integrations

##### AWS-SNS-Listener

- Added a **Request processing mode** parameter that determines whether incoming SNS notifications are processed at the same time (*concurrent*) or one at a time (*sequential*, the default), in order to avoid duplicate notifications during bursts of traffic.
- Fixed a bug where sample events for the mapping wizard were duplicated and re-written on every notification.
- Fixed a bug where AWS SNS notifications without a **Subject** field caused signature verification to fail with an internal error and the incident to be silently dropped.
