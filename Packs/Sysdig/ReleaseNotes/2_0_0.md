
#### Classifiers

##### Sysdig Mapper Runtime Event

- Updated the incoming mapper to match the Sysdig Secure V2 webhook payload structure (`entities[0].policyEvents[0]` and `eventLabels`).
- Added a severity transformer mapping Sysdig severity (0=highest, 7=lowest) to XSOAR severity (1-4).
- Added timestamp conversion from microseconds to ISO date format.
- Removed the mapping of `Sysdig Agent ID` and `Sysdig Customer ID` from the webhook payload as they are not present in the V2 format. These are now resolved automatically via the `sysdig-agent-info-get` command.

#### Integrations

##### Sysdig Response Actions

- Added the **sysdig-agent-info-get** command: resolves a host MAC address to its Sysdig agent details (agent ID, customer ID, hostname) via the `/api/agents/connected` endpoint. Results are cached per MAC address with a 1-hour TTL.
- Added the **sysdig-customer-info-get** command: retrieves the Sysdig customer ID and name via `/api/users/me`. Results are cached with a 1-hour TTL.
- Added support for 22 new response action types in the **execute-response-action** command (DELETE_POD, ROLLOUT_RESTART, ISOLATE_NETWORK, GET_LOGS, IAM_QUARANTINE, CAPTURE, CLOUD_VOLUME_SNAPSHOT, FETCH_CLOUD_LOGS, and more) with per-action parameter validation.

#### Layouts

##### Sysdig Incident

#### Playbooks

##### Sysdig Trigger System Capture

- Added automatic agent enrichment: the playbook now calls `sysdig-agent-info-get` to resolve the host MAC address to agent ID, customer ID, and hostname before triggering a system capture.
- Updated playbook validation to require only the MAC address (minimum field needed to resolve agent info via API).
- Updated `agent_id`, `customer_id`, and `host_name` to be sourced from the Sysdig API response instead of the webhook payload.
