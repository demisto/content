
#### Integrations

##### Sysdig Response Actions

- Added **sysdig-get-agent-info** command: resolves a host MAC address to its Sysdig agent details (agent ID, customer ID, hostname) via the `/api/agents/connected` endpoint. Results are cached per MAC address with a 1-hour TTL.
- Added **sysdig-get-customer-info** command: retrieves the Sysdig customer ID and name via `/api/users/me`. Results are cached with a 1-hour TTL.

#### Classifiers

##### Sysdig Mapper Runtime Event

- Rewrote the incoming mapper to match the real Sysdig Secure V2 webhook payload structure (`entities[0].policyEvents[0]` and `eventLabels`).
- Added severity transformer mapping Sysdig severity (0=highest, 7=lowest) to XSOAR severity (1-4).
- Added timestamp conversion from microseconds to ISO date format.
- The fields `Sysdig Agent ID` and `Sysdig Customer ID` are no longer mapped from the webhook payload as they are not present in the V2 format. These are now resolved automatically via the `sysdig-get-agent-info` command.

#### Incident Fields

- **New:** Sysdig Event ID, Sysdig Container Name, Sysdig Container Image, Sysdig Container Image Tag, Sysdig Container Image ID, Sysdig Host ID, Sysdig Agent Version, Sysdig Agent Variant, Sysdig Rule Name, Sysdig Rule Type, Sysdig Policy ID, Sysdig Policy Name, Sysdig Policy Version, Sysdig Falco Output, Sysdig Event URL, Sysdig Source, Sysdig Origin, Sysdig Aggregated, Sysdig K8s Cluster Name, Sysdig K8s Namespace, Sysdig K8s Pod Name, Sysdig K8s Node Name, Sysdig K8s Workload Name, Sysdig K8s Workload Type, Sysdig AWS Account ID, Sysdig AWS Region, Sysdig AWS Instance ID, Sysdig Process Name, Sysdig Process Cmdline, Sysdig Process CWD, Sysdig Process SHA256, Sysdig Process Parent Name, Sysdig Event Type, Sysdig Username.

#### Layouts

##### Sysdig Incident

- Reorganized the Incident Info tab into logical sections: Sysdig Event Info, Container & Host, Kubernetes Context, Cloud Context, Process Details, and Falco Rule Output.
- Added Legacy Sysdig Fields section for backward compatibility (Agent ID, Customer ID).

#### Playbooks

##### Sysdig Trigger System Capture

- Added automatic agent enrichment: the playbook now calls `sysdig-get-agent-info` to resolve the host MAC address to agent ID, customer ID, and hostname before triggering a system capture.
- Validation simplified to require only MAC address (minimum field needed to resolve agent info via API).
- `agent_id`, `customer_id`, and `host_name` are now sourced from the Sysdig API response instead of the webhook payload.

## Breaking Changes

- **Sysdig Agent ID** and **Sysdig Customer ID** are no longer mapped from the webhook payload. The Sysdig Secure V2 webhook format does not include these fields. They are now resolved via the `sysdig-get-agent-info` API command during playbook execution. If you have custom automations reading these fields directly from the incident, update them to run after the enrichment playbook populates them.
