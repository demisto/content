#### Integrations

##### AWS SNS SMS Communication

- **CRITICAL FIX: Task ID now properly saved in entitlements (v1.0.24)**
  - Fixed `extract_entitlement_from_message()` to extract full entitlement string format: `GUID@incident_id|task_id`
  - Previously only extracted GUID, then rebuilt as `GUID@incident_id` (dropping task ID)
  - Now regex captures full format including optional task ID: `@incident_id|task_id`
  - `send_notification_command` now uses extracted full string instead of rebuilding
  - Fixes "Task ID: N/A" error in inject-reply command output
  - Enables proper playbook task resumption when SMS replies are received

- **Technical details:**
  - New regex pattern: `([a-f0-9-]{36})@(\d+)(?:\|([^\s]+))?`
  - Captures: GUID, incident ID, and optional task ID
  - SMSAskUser creates: "Question - Reply Yes or No: GUID@69|5"
  - Integration now stores: "GUID@69|5" (previously "GUID@69")
  - Task closure now works correctly via `demisto.handleEntitlementForUser()`
