#### Integrations

##### Gurucul-GRA

- Added **Fetch type** parameter (`Incidents` | `Alerts`). New imports use GRA Incidents or Alerts APIs; Cases are no longer fetched.
- Added incident commands: ***gra-incidents***, ***gra-incident-action***, ***gra-incident-action-anomaly***, ***gra-incidents-anomaly***.
- Added alert commands: ***gra-alerts***, ***gra-alert-get***, ***gra-alert-action***, ***gra-alert-update-history***.
- Added thin War Room alert commands: ***gra-alert-comment***, ***gra-alert-assign***, ***gra-alert-in-progress***.
- Documented ***gra-alert-action*** closeAlert subStatus enums (Incident: True Positive, Benign Positive, Policy Violation; Not An Incident: Model Review, Disable Model, False Positive).
- Existing ***gra-case-*** commands remain for already-imported GRACase incidents.
- Incident and Alert fetch now use an ID cursor after the first bootstrap (`maxIncidentId` / `maxAlertId`). The time window from **First fetch time** is used only when no cursor exists yet.
- On upgrade from Case fetch, an existing `maxCaseId` last-run cursor is reused as `maxIncidentId` when the Incident cursor is missing (avoids re-bootstrapping with **First fetch time**).
- **Upgrade note:** Before re-enabling fetch after updating to this version, disable the old Cases instance fetch, set **Fetch type** / Classifier / Mapper / Incident type for **Incidents** (`GRAIncident-Classifier`, `GRAIncident-Mapper`, `GRAIncident`), then re-enable. See the Gurucul-GRA integration README section *Upgrading from Case fetch (2.1.0)*.

#### Incident Types / Layouts / Fields / Classifiers

- Added **GRAIncident** and **GRAAlert** types, layouts, fields, classifiers, and incoming mappers.
- Added **GRA Alert Assignee Ids** field and mapper mapping from `assigneeIds`.
- Legacy **GRACase** artifacts are unchanged.

#### Scripts

- Added **GRAUpdateIncidentStatus**, **GRAIncidentAnomaliesDisplay**.
- Added **GRAUpdateAlertStatus**, **GRAAlertHistoryDisplay**, **GRAAlertAnalyticalFeatureDisplay**, **GRAAlertClassifierDisplay**.
- Updated **GRAAnalyticalFeatureDisplay** to support Incident anomaly fields as well as Case.
- **GRAAlertHistoryDisplay** returns markdown tables with Action/Comment/Date (formatted); **GRAAlertAnalyticalFeatureDisplay** shows Analytical Feature/Values (Count omitted; frequencies are in Values).
- **GRAAlertClassifierDisplay** parses classifier JSON into a Classifier/Detail table.

#### Layouts

- **GRAAlertLayout**: Alert Details includes status, severity, risk score, sub-status, resource, classifier, assignee type, assignee ids, and related triage fields; Classifiers dynamic section; removed leftover Case-clone Extra and Investigation Data sections.

#### Playbooks

- Added **GRAIncident** and **GRAAlert** playbooks.
- **GRAAlert** playbook prints a War Room triage summary (anomaly, entity, status, severity, resource, assignee, link).
