#### Integrations

##### Google Cloud Platform

- Added support for **Cortex XSOAR** and **Cortex XSIAM** marketplace deployments. The integration can now be configured with a GCP Service Account private key JSON and a default *GCP Project ID* for use outside of Cortex Cloud.
- Updated the *Service Account Private Key (JSON)* parameter description to clarify that it is required for Cortex XSOAR and Cortex XSIAM, and optional on Cortex Cloud.
- Added the *GCP Project ID* integration parameter, used by the test button on Cortex XSOAR and Cortex XSIAM to validate credentials.
- Updated the ***test-module*** command to call the Cloud Resource Manager `testIamPermissions` API directly when running on Cortex XSOAR or Cortex XSIAM, validating that the provided service account credentials are functional.
- Updated the integration description to cover all three supported platforms.
- Updated the *project_id* arguments in all commands to `required: false` with `required:platform: true`, making the argument optional on Cortex XSOAR and Cortex XSIAM (where it can be inferred from the service account JSON) while remaining required on Cortex Cloud (platform).
