
#### Modeling Rules

##### Fortinet FortiGate Modeling Rule

Fixed an issue where FortiGate events were incorrectly assigned the **iOS** OS family. The `xdm.source.host.os_family` and `xdm.target.host.os_family` mappings matched the substring `ios` inside `FortiOS`, so the operating system family for FortiGate (FortiOS) events was set to iOS. The mapping now uses `lowercase()` and matches the operating system name exactly.
