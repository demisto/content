#### Scripts

##### GenericPollingScheduledTask

- Fixed an issue where generic polling could complete the blocking task prematurely when the polling data was not yet present in the context on the first iteration (for example, when the first scheduled poll runs before the polling command has populated the sub-playbook context). An empty context query result is no longer treated as "all IDs finished" in either polling flow; polling now continues (bounded by the configured timeout in the legacy flow, or the absolute end time in the GUID-based flow) until the polling data is available or the run is bounded out.

##### ScheduleGenericPolling

- Fixed an issue where the ``unified_platform`` (Cortex/XSIAM) platform used the legacy polling flow instead of the GUID-based flow. Polling on ``unified_platform`` now uses the GUID-based flow, which stops on an absolute end time.
