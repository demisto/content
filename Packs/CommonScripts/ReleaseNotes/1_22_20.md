
#### Scripts

##### ScheduleGenericPolling

- Fixed an issue where the polling task could terminate prematurely (before the configured timeout was reached) by scheduling the polling entry as a recurring entry that spans the entire timeout window, instead of relying on the task re-scheduling itself on every iteration.

##### GenericPollingScheduledTask

- Fixed an issue where the polling task could terminate prematurely (before the configured timeout was reached) by no longer re-scheduling itself each iteration, since recurrence is now owned by the single scheduled entry created by **ScheduleGenericPolling**.
