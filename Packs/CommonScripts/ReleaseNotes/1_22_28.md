
#### Scripts
##### ScheduleGenericPolling

- Fixed an issue where the polling task could terminate prematurely (before the configured timeout was reached) by scheduling the polling entry with an absolute end time, giving the polling task a reliable, run-independent deadline.

##### GenericPollingScheduledTask

- Fixed an issue where the polling task could complete on its first run when the pending IDs were not yet present in the context. The task now continues polling until the items reach their terminal state or the configured timeout is reached, instead of treating a not-yet-populated context as completion.
