
#### Scripts

##### GenericPollingScheduledTask

- Hardened error handling so that a transient failure of the polling command (for example a momentary API/network error or a temporarily unavailable polling integration) no longer completes the polling task and advances the parent playbook prematurely. While the polling window is still open the error is logged and the next scheduled recurrence retries; the task is only completed once the polling window has elapsed.
- Enabled the scheduled-entry GUID flow (using `stopScheduleEntry`) on Cortex XSIAM, so the recurring polling entry is cleanly cancelled when polling completes.
