
#### Triggers Recommendations

##### New: Allow Access by Trusted Services

- New: Added a new triggers recommendation- Allow Access by Trusted Services that This CLI command requires 'Microsoft.Storage/storageAccounts/*' permission. Successful execution will enable Azure accounts to bypass 'Trusted Microsoft Services'.

##### New: Assign Managed Identity to Function App

- New: Added a new triggers recommendation- Assign Managed Identity to Function App that This CLI command requires 'Microsoft.Web/sites/Write' permission. Successful execution sets managed service identity in Azure Function App, that makes the app more secure by eliminating secrets from the app, such as credentials in the connection strings.

##### New: Assign Managed Identity to Web App

- New: Added a new triggers recommendation- Assign Managed Identity to Web App that This CLI command requires 'Microsoft.Web/sites/{app_name}/config/*' permission. Successful execution sets managed service identity in App Service, that makes the app more secure by eliminating secrets from the app, such as credentials in the connection strings.

##### New: Disable Network Access to Storage Account

- New: Added a new triggers recommendation- Disable Network Access to Storage Account that This CLI command requires 'Microsoft.Storage/storageAccounts/*' permission. Successful execution will disable default network access rule for Storage Accounts.'.

##### New: Disable Public & Private Access to VM Disk

- New: Added a new triggers recommendation- Disable Public & Private Access to VM Disk that This CLI command requires 'Microsoft.Compute/disks/write' permission. Successful execution will disable public network access and sets the network access policy to DenyAll for the specified disk, blocking external access. 
Note: As a security best practice, we are disabling public and private access to the disk. However, you can choose between the following options based on your security requirements:
- Disable public access and enable private access
- Disable public and private access

##### New: Disable Public Access on RDS

##### New: Disable Remote Debugging on Azure App

##### New: Enable Automatic Backup for RDS dB

##### New: Enable Azure App Service Auth

- New: Added a new triggers recommendation- Enable Azure App Service Auth that This CLI command requires 'Microsoft.Web/sites/{app_name}/config/authsettings/*' permission. Successful execution will enable app service authentication, that can prevent anonymous HTTP requests from reaching the API app, or authenticate those that have tokens before they reach the API app.

##### New: Set AMI to Private

- New: Added a new triggers recommendation- Set AMI to Private that This CLI command requires 'ec2:ModifyImageAttribute' permission. Successful execution will make this AMI's access to private. This will ensure that only Owner has full privileges.

##### New: Set Function App HTTP Version to 2.0

- New: Added a new triggers recommendation- Set Function App HTTP Version to 2.0 that This CLI command requires 'Microsoft.Web/sites/config/Write' permission. Successful execution sets HTTP version to 2.0, which has additional performance improvements on the head-of-line blocking problem of old HTTP version, header compression, and prioritization of requests.
