
#### Triggers Recommendations

Added the following new triggers recommendations:
- **Allow Access by Trusted Services**: Requires the 'Microsoft.Storage/storageAccounts/*' permission. Running the command for this recommendation enables Azure accounts to bypass 'Trusted Microsoft Services'.
- **Assign Managed Identity to Function App**: Requires the 'Microsoft.Web/sites/Write' permission. Running the command for this recommendation sets a managed service identity in the Azure Function App to eliminate secrets (such as credentials in connection strings) and increase security.
- **Assign Managed Identity to Web App**: Requires the 'Microsoft.Web/sites/{app_name}/config/*' permission. Running the command for this recommendation sets a managed service identity in App Service to eliminate secrets (such as credentials in connection strings) and increase security.
- **Disable Network Access to Storage Account**: Requires the 'Microsoft.Storage/storageAccounts/*' permission. Running the command for this recommendation disables the default network access rule for Storage Accounts.
- **Disable Public & Private Access to VM Disk**: Requires the 'Microsoft.Compute/disks/write' permission. Running the command for this recommendation disables public network access and sets the network access policy to **DenyAll** for the specified disk, blocking external access. Options include disabling public access while enabling private access, or disabling both public and private access.
- **Disable Public Access on RDS**: Requires the 'rds:ModifyDBInstance' permission. Running the command for this recommendation disables the publicly accessible configuration of AWS RDS database instances.
- **Disable Remote Debugging on Azure App**: Requires the 'Microsoft.Web/sites/config/Write' permission. Running the command for this recommendation disables remote debugging on Azure App Service apps.
- **Disable Storage Account Cross Tenant Replication**: Requires the 'Microsoft.Storage/storageAccounts/*' permission. Running the command for this recommendation disables cross-tenant replication in Storage Accounts.
- **Enable Automatic Backup for RDS DB**: Requires the 'rds:ModifyDBInstance' permission. Running the command for this recommendation enables the automatic backup settings for RDS instances.
- **Enable Azure App Service Auth**: Requires the 'Microsoft.Web/sites/{app_name}/config/authsettings/*' permission. Running the command for this recommendation enables App Service authentication to prevent anonymous HTTP requests from reaching the API app, or authenticates requests containing tokens before they reach the API app.
- **Enable CloudTrail Log Validation**: Requires the 'cloudtrail:UpdateTrail' permission. Running the command for this recommendation enables log file validation for the specified CloudTrail.
- **Enable CloudTrail Logging**: Requires the 'cloudtrail:StartLogging' permission. Running the command for this recommendation enables logging for the respective CloudTrail.
- **Enable Copy Tags on RDS Snapshot**: Requires the 'rds:ModifyDBInstance' permission. Running the command for this recommendation enables 'copy tags to snapshots' for the respective RDS instance.
- **Enable Deletion Protection on RDS**: Requires the 'rds:ModifyDBInstance' permission. Running the command for this recommendation enables deletion protection for the reported AWS RDS instance.
- **Enable GCP Bucket Versioning**: Requires the 'storage.buckets.update' permission. Running the command for this recommendation enables GCP Storage log buckets versioning.
- **Enable GKE Cluster Intra-node Visibility**: Requires the 'container.clusters.update' permission. Running the command for this recommendation enables intra-node visibility for the GCP Kubernetes Engine cluster to expose flow logs for traffic between Pods.
- **Enable IAM Authentication on RDS**: Requires the 'rds:ModifyDBInstance' permission. Running the command for this recommendation enables IAM Authentication for the reported AWS RDS instance.
- **Enable IAM Authentication on RDS Cluster**: Requires the 'rds:ModifyDBCluster' permission. Running the command for this recommendation enables IAM Authentication for the reported AWS RDS cluster.
- **Enable Master Authorized Networks on GKE**: Requires the 'container.clusters.update' permission. Running the command for this recommendation enables the Master authorized network for the GCP Kubernetes Engine cluster by blocking untrusted non-GCP source IPs from accessing the Kubernetes master through HTTPS.
- **Enable RDS Auto Upgrade**: Requires the 'rds:ModifyDBInstance' permission. Running the command for this recommendation enables minor upgrades for the RDS database instance.
- **Enable RDS Cluster Deletion Protection**: Requires the 'rds:ModifyDBCluster' permission. Running the command for this recommendation enables deletion protection for the reported AWS RDS cluster.
- **Enable S3 Versioning**: Requires the 's3:PutBucketVersioning' permission. Running the command for this recommendation enables Object Versioning for the respective S3 bucket.
- **Enable Soft Delete on Blob**: Requires the 'Microsoft.Storage/storageAccounts/blobServices/write' permission. Running the command for this recommendation enables soft delete for blobs on Azure Storage accounts. By default, the delete retention period is configured to 30 days, which can be modified by cloning the policy.
- **Make GCP Bucket Private**: Requires the 'storage.buckets.getIamPolicy' and 'storage.buckets.setIamPolicy' permissions. Running the command for this recommendation revokes 'allUsers' and 'allAuthenticatedUsers' permission access on GCP Storage buckets.
- **Remove GCP Bucket AllAuthenticatedUser Access**: Requires the 'storage.buckets.getIamPolicy' and 'storage.buckets.setIamPolicy' permissions. Running the command for this recommendation revokes 'allAuthenticatedUsers' permission access on GCP Storage buckets.
- **Remove GCP Bucket AllUsers Access**: Requires the 'storage.buckets.getIamPolicy' and 'storage.buckets.setIamPolicy' permissions. Running the command for this recommendation revokes 'allUsers' permission access on GCP Storage buckets.
- **Set AMI to Private**: Requires the 'ec2:ModifyImageAttribute' permission. Running the command for this recommendation configures the AMI access level to private to ensure only the owner retains full privileges.
- **Set Function App HTTP Version to 2.0**: Requires the 'Microsoft.Web/sites/config/Write' permission. Running the command for this recommendation sets the HTTP version to 2.0 to resolve head-of-line blocking issues, introduce header compression, and enable request prioritization.
- **Set Function App Min TLS Version**: Requires the 'Microsoft.Web/sites/config/Write' permission. Running the command for this recommendation sets the Web App TLS encryption version to the latest standard (i.e., **TLS 1.2**).
- **Set GCP Bucket Access to Uniform**: Requires the 'storage.buckets.update' permission. Running the command for this recommendation sets uniform bucket-level access in GCP Storage buckets. Note that once uniform bucket-level access has been enabled on a bucket for 90 consecutive days, it can no longer be disabled.
- **Set RDS Cluster Snapshot to Private**: Requires the 'rds:ModifyDBClusterSnapshotAttribute' permission. Running the command for this recommendation resets the RDS Cluster snapshot Access Control List (ACL) to private to ensure only the owner retains full privileges.
- **Set RDS Snapshot to Private**: Requires the 'rds:ModifyDBSnapshotAttribute' permission. Running the command for this recommendation resets the RDS snapshot Access Control List (ACL) to private to ensure only the owner retains full privileges.
- **Set Secure Transport for MySQL**: Requires the 'Microsoft.DBforMySQL/flexibleServers/configurations/write' permission. Running the command for this recommendation enables SSL connections on your Azure MySQL database flexible server.
- **Set Snapshot to Private**: Requires the 'ec2:ModifySnapshotAttribute' permission. Running the command for this recommendation configures the EBS snapshot access level to private to ensure only the owner retains full privileges.
- **Set Storage Account to HTTPS only**: Requires the 'Microsoft.Storage/storageAccounts/*' permission. Running the command for this recommendation enables secure transfer for the specified storage account.
- **Set Web App Min TLS Version**: Requires the 'Microsoft.Web/sites/{app_name}/config/*' permission. Running the command for this recommendation sets the Web App TLS encryption version to the latest standard (i.e., **TLS 1.2**).
- **Set Webapp HTTP Version to 2.0**: Requires the 'Microsoft.Web/sites/{app_name}/config/*' permission. Running the command for this recommendation sets the HTTP version to 2.0 to resolve head-of-line blocking issues, introduce header compression, and enable request prioritization.
- **Update Azure Monitor Log Retention Period**: Requires the 'Microsoft.Insights/LogProfiles/[Read, Write, Delete]' permission. Running the command for this recommendation updates the Azure Monitor log profile retention policy to 365 days.
