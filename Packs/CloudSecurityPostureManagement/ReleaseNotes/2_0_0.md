
#### Triggers Recommendations

##### New: Trigger - AWS Network Exposure

Added the ***Trigger - AWS Network Exposure*** triggers recommendation that recommends the **AWS Network Exposure - Auto-remediate** playbook to automatically remediate the network exposure of AWS resources by updating the AWS asset with a new security group which is a copy of the original but without the offending security rule that causes public exposure.
<~XSIAM> (Available from Cortex XSIAM V3.5).</~XSIAM>

##### New: Trigger - Azure Network Exposure

Added the ***Trigger - Azure Network Exposure*** triggers recommendation that recommends the **Azure Network Exposure - Auto-remediate** playbook. This playbook remediates Azure VM network exposure by updating the Network Security Group (NSG). It adds a high-priority Deny rule to block public access and an Allow rule to permit internal IP traffic.
<~XSIAM> (Available from Cortex XSIAM V3.5).</~XSIAM>

##### New: Trigger - GCP Network Exposure

Added the ***Trigger - GCP Network Exposure*** triggers recommendation that recommends the **GCP Network Exposure - Auto-remediate** playbook. This playbook remediates the network exposure of GCP resources by adding firewall rules to deny public internet access on specified ports and allow access from internal IPs.
<~XSIAM> (Available from Cortex XSIAM V3.5).</~XSIAM>

#### Playbooks

##### New: AWS Network Exposure - Auto-remediate

This playbook remediates AWS Network Exposure detections, specifically Amazon EC2 instances with management ports exposed to the public internet. It updates the AWS asset with a new security group that is a copy of the original but excludes the security rule causing public exposure.

##### New: Azure Network Exposure - Auto-remediate

This playbook remediates Azure Network Exposure detections where virtual machines have management ports exposed to the public internet. It updates the Network Security Group (NSG) by adding a high-priority Deny rule to block public access and an Allow rule to permit internal traffic.

##### New: GCP Network Exposure - Auto-remediate

This playbook automatically remediates Google Compute Engine instances with management ports exposed  to the internet. It adds GCP firewall rules to deny public access on specified ports while allowing internal IP traffic. It also uses network tags to apply rules to resources or adds the required tag to assets if the rule already exists.
