
#### Integrations

##### Tenable.sc

- Added extraction of all supported host fields for the assets (hosts) collection, including `fqdnIndex`, `netBiosWorkgroup`, and `repID`.
- Replaced the in-place `hostUUID`/`vulnUUID` generation with dedicated composite-key columns, so the raw `hostUUID`/`vulnUUID` values returned by Tenable.sc are preserved untouched. A `hostUniqueness` column (derived from `repositoryID`, `ip`, `dnsName`) and a `vulnUniqueness` column (derived from `repositoryID`, `ip`, `port`, `protocol`, `pluginID`) are now added to vulnerabilities, and a `hostUniqueness` column (derived from `repositoryID`, `ipAddress`, `dns`) is now added to assets, so findings can be correlated even for network scans that do not include native identifiers.
- Added snapshot-based ingestion for the vulnerabilities collection, matching the assets collection, so complete and incomplete snapshots are tracked correctly.
