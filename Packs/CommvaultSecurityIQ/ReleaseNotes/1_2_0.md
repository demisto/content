
#### Integrations

##### Commvault Cloud

- Added authentication enforcement for the long-running webhook listener. Inbound webhook requests are now rejected with HTTP 401 unless they present the configured listener credentials.
- Added a new *Webhook Listener Credentials* parameter (username and password) used to authenticate inbound webhook requests. This parameter is required when the *Forwarding Rule* is set to **Webhook**. Existing Webhook instances must configure these credentials on both the integration instance and the Commvault Cloud webhook sender. Instances using *Fetch Incidents* or the *Syslog* forwarding rule are not affected.
- Updated the Docker image to: *demisto/commvault:1.0.0.9311457*.
