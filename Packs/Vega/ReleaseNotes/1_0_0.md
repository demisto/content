#### Integrations

##### Vega

- Mirrored Vega incident comments now appear in the War Room as plain comment notes instead of verbose tables.
- Added incoming and outgoing comment mirroring support through `get-remote-data`, `get-modified-remote-data`, and `update-remote-system`.
- Normalized `verdictReasoning` to display `N/A` when null.
- Mapped the Vega alert UUID (`id`) to the **Alert ID** incident field and use it to fetch alert events.
- Fixed alert events detection: rows containing `cid`/`eid` return "No alert events found"; all other shapes are rendered normally.
- Renamed `vega-fetch-alert-events` integration command to `vega-get-alert-events` and the layout script to `vega-fetch-alert-events`.
- Fetched Vega alert comments from `getAlerts` and render them in the Vega Comments incident field.
- Added `assignees` argument to the ***vega-update-alert*** command to match the Vega UpdateAlerts API.
- Added `assignee_emails` argument to the ***vega-update-incident*** command to match the Vega UpdateIncidents API.
- Renamed the primary update command arguments to `alert_ids` and `incident_ids` to reflect multi-ID support. `alert_id` and `incident_id` remain as hidden backward-compatible aliases.
- Added automatic retries for transient Vega API failures such as HTTP 504 Gateway Timeout during fetch and other requests.
- Fetch incidents now skips only the affected entity type on transient API errors instead of failing the entire fetch job, and uses smaller pagination pages to reduce timeout risk.
- Fixed the ***vega-update-incident*** GraphQL mutation to stop requesting `severity` from the `UpdatedIncident` response type, which the Vega API does not expose.

#### Incident Fields

##### Vega Verdict Reasoning

- Restored the field type to long text so analysts can enter multiline verdict reasoning.
- Field edits in XSOAR are pushed to Vega through outgoing mirroring via `update-remote-system`.
- Fixed Vega Incident verdict reasoning mirroring by always loading incident details and ignoring `N/A` placeholders when extracting reasoning.

#### Layouts

##### Vega Alert / Vega Incident

- Added Vega Verdict Reasoning below Vega Verdict in the layout properties section.
- Added Vega Comments section to the Vega Alert layout, mirroring the Vega Incident comments display.

#### Incident Fields

##### Vega Comments

- Extended the field to Vega Alert incidents and mapped alert comments from the Vega API.

#### Classifiers

##### Vega Mapper

- Added Vega Comments mapping for incoming Vega Alert data.
