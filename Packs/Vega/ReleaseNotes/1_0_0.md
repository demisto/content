#### Integrations

##### Vega

- Initial release of the Vega integration for Cortex XSOAR.
- Updated the mirrored Vega incident comments to now appear in the War Room as plain comment notes instead of verbose tables.
- Added incoming and outgoing comment mirroring support through `get-remote-data`, `get-modified-remote-data`, and `update-remote-system`.
- Updated `verdictReasoning` to display `N/A` when null.
- Updated the mapping for Vega alert UUID (`id`) to the **Alert ID** incident field and use it to fetch alert events.
- Fixed alert events detection: rows containing `cid`/`eid` return "No alert events found"; all other shapes are rendered normally.
- Updated the `vega-fetch-alert-events` integration command name to `vega-get-alert-events` and the layout script to `vega-fetch-alert-events`.
- Updated the fetch process to get Vega alert comments from `getAlerts` and render them in the Vega Comments incident field.
- Added `assignees` argument to the ***vega-update-alert*** command to match the Vega UpdateAlerts API.
- Added `assignee_emails` argument to the ***vega-update-incident*** command to match the Vega UpdateIncidents API.
- Updated the primary update command arguments to `alert_ids` and `incident_ids` to reflect multi-ID support. `alert_id` and `incident_id` remain as hidden backward-compatible aliases.
- Added automatic retries for transient Vega API failures such as HTTP 504 Gateway Timeout during fetch and other requests.
- Improved fetch incidents to skip only the affected entity type on transient API errors instead of failing the entire fetch job, and to use smaller pagination pages to reduce timeout risk.
- Fixed the ***vega-update-incident*** GraphQL mutation to stop requesting `severity` from the `UpdatedIncident` response type, which the Vega API does not expose.
- Updated the fetch process to load alert events together with `getAlerts`, so the Alert Events layout section is populated during ingest without a separate `getAlertsEvents` call.
- Removed the `vega-fetch-alert-events` layout widget script; alert events are shown statically from the **Vega Alert Events** incident field.
- Updated incoming mirror sync to set **dbotMirrorDirection** and **dbotMirrorInstance** directly on each `get-remote-data` response so open investigations refresh mirror direction every mirror cycle.
- **vega-get-alert-events**
  - Added the integration command to fetch alert events from the Vega API.

#### Incident Fields

##### Vega Verdict Reasoning

- Updated the field type to long text so analysts can enter multiline verdict reasoning.
- Updated outgoing mirroring to push field edits in Cortex XSOAR to Vega via `update-remote-system`.
- Fixed Vega Incident verdict reasoning mirroring by always loading incident details and ignoring `N/A` placeholders when extracting reasoning.

##### Vega Comments

- Updated the field to support Vega Alert incidents and mapped alert comments from the Vega API.

#### Classifiers

##### Vega Mapper

- Updated the incoming mapper to stop mapping raw Vega alert and incident JSON into incident labels, preventing duplicate field values from appearing in the War Room.
- Added mapping from Vega alert `id` to the **Alert ID** incident field.
- Added Vega Comments mapping for incoming Vega Alert data.
- Added **dbotMirrorDirection** mapping for Vega Alert investigations so incoming mirror sync updates mirror direction for open alerts, matching Vega Incident behavior.

#### Layouts

##### Vega Alert

- Added Vega Verdict Reasoning below Vega Verdict in the layout properties section.
- Added Vega Comments section to the Vega Alert layout, mirroring the Vega Incident comments display.
- Removed the dynamic Alert Events layout section and replaced it with a static **Vega Alert Events** field populated at ingest.

##### Vega Incident

- Added Vega Verdict Reasoning below Vega Verdict in the layout properties section.
