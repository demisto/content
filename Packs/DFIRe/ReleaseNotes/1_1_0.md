#### Integrations

##### DFIRe

- Expanded the integration to track the upstream DFIRe API. Command count grew from 29 to 75. Highlights:
  - **Extended case-create/update**: new fields `notes`, `incident_category`, `outcome_verdict`, `investigators`, `viewers`, `attributes`, plus an explicit `create_slack_channel` arg that **defaults to `false`** so SOAR-driven case creation does not silently spawn Slack channels.
  - **Extended case-list**: new filters `created_at_gte`, `created_at_lte`, `lead_investigator`, `status_in`, `ordering`.
  - **Extended case-indicator-add**: new fields `source`, `valid_until`, `publish`.
  - **Extended indicator-list**: new filters `parent`, `ordering`.
- Added IOC commands: `dfire-ioc-extract`, `dfire-indicator-check`, `dfire-indicator-enrich`, `dfire-indicator-enrichment-list`, `dfire-indicator-publish` / `unpublish` / `revoke` / `unrevoke` / `decompose` / `add-tags`, `dfire-indicator-correlated-list`, and bulk operations: `bulk-classify`, `bulk-confidence`, `bulk-tag`, `bulk-tlp`, `bulk-publish`, `bulk-revoke`, `bulk-delete`.
- Added case-side automation: `dfire-case-generate-summary`, `dfire-case-chat`, `dfire-case-update-report`, `dfire-case-can-report-list` / `generate`, `dfire-case-investigation-report-get` / `generate` / `finalize` / `ready-for-qa`, `dfire-case-timeline-change-phase`.
- Added case todo commands: `dfire-case-todo-list`, `get`, `assign`, `note-set`, `attach-runbook`, `detach-runbook`.
- Added case timer (SLA) commands: `dfire-case-timer-list`, `get`, `complete`, `reset`.
- Added convenience lookups: `dfire-case-get-by-number`, `dfire-item-resolve-short-id`.
- Added reference-data list commands for picklists: `dfire-incident-category-list`, `dfire-incident-phase-list`, `dfire-outcome-verdict-list`, `dfire-project-list`, `dfire-runbook-list`, `dfire-group-list`.
- Bumped pack version to **1.1.0**.
