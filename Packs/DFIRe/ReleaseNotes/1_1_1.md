#### Integrations

##### DFIRe

- Fixed `dfire-item-get`: `item_id` is now treated as the UUID string the API expects (previously the integration coerced it to an integer, so every call failed).
- Fixed `dfire-case-generate-summary`: switched from POST to GET to match the API, and stopped trying to parse the plain-text response body as JSON. The summary now lands in `DFIRe.CaseSummary.summary`.
- Fixed `dfire-case-todo-list` and `dfire-case-todo-get`: the API has no GET endpoint for todos, so both commands now fetch the case and read its `todo_checklist` field. Behaviour from the playbook author's perspective is unchanged.
- Fixed `dfire-case-todo-note-set`: switched from POST to PUT to match the API.
- Fixed `dfire-timeline-create`: the API requires `event_datetime`; the integration now defaults it to "now" when not provided.
- Fixed `dfire-case-investigation-report-generate` and `dfire-case-investigation-report-ready-for-qa`: both endpoints operate per section, so a new required `section_id` argument has been added.
- Kept `todo_id` as a string throughout the todo sub-commands so non-numeric IDs are accepted.
