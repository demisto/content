
#### Integrations

##### CohesityHelios

- Migrated ransomware alerts API from v1 to v2 (`/v2/mcm/alerts`).
- Updated alert filtering to use `alertName` instead of `alertCode`.
- Commands `cohesity-helios-ignore-anomalous-object` and `cohesity-helios-restore-latest-clean-snapshot` now use `alert_id` instead of `object_name`.
- Added `_api_request` wrapper with debug logging and improved error handling.
- Updated command outputs with context fields: `cluster_id`, `cluster_name`, `entity_id`, and `job_id`.
- Removed unused command output context fields: `anomalous_object_name`, `anomalous_object_env`, and `anomaly_strength`.

#### Incident Fields

##### Cohesity Helios Alert Id

- Updated the **Cohesity Helios Alert Id** incident field and set it as unsearchable.

##### Cohesity Helios Object Id

- Updated the **Cohesity Helios Object Id** incident field and set it as unsearchable.

#### Classifiers

##### CohesityHelios Incident Classifier

- Updated classification to use `DataIngestAnomalyAlert` alert name.
- Updated transformer to use `alertDocument.alertName`.

#### Mappers

##### Cohesity Helios Incoming mapper

- Updated the **Cohesity Helios Incoming mapper** mappings for Alert Id and Object Id, and removed deprecated fields.

#### Layouts

##### Cohesity Helios Ransomware Incident Layout

- Updated the **Cohesity Helios Ransomware Incident Layout** to use the new Alert Id and Object Id fields.
- Updated layout action buttons to pass `alert_id` instead of `object_name`.
