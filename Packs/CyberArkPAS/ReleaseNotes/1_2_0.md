
#### Integrations

##### CyberArk Identity Security Platform

- Added support for `fetch-assets`, which collects identity directory snapshots from the CyberArk Identity Cloud Directory and lands them in dedicated XSIAM datasets per source. Configurable via the new **Fetch Assets** checkbox and the **Directory Data Collection** multi-select (Users, Groups, Roles, Applications). Snapshots are fetched via the CyberArk `/Redrock/Query` API and pushed using the snapshot data type so deletions on the CyberArk side are reflected on the next fetch cycle.
- Added the following datasets:
  - *cyberark_users_raw* — directory user snapshots.
  - *cyberark_groups_raw* — directory group snapshots.
  - *cyberark_roles_raw* — directory role snapshots.
  - *cyberark_applications_raw* — application snapshots.
- Added the following commands for manual debug fetch (with optional `should_push_assets`):
  - ***cyberark-isp-get-users***
  - ***cyberark-isp-get-groups***
  - ***cyberark-isp-get-roles***
  - ***cyberark-isp-get-applications***
- Updated the Docker image to: *demisto/python3:3.12.12.6391686*.
