
#### Integrations

##### CrowdStrike Falcon

- Fixed an issue where the ***cs-falcon-apply-quarantine-file-action*** command failed with `400 Bad Request - Validation error` when the **filename** (or other list-typed) search argument was used. The FQL query parameter is now built correctly for both single- and multi-value inputs, and the command now returns a clear "no files matched" message instead of issuing a malformed PATCH when the search yields no results.
