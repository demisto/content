
#### Integrations

##### Akamai WAF SIEM

- Fixed an issue with out-of-memory failures that could occur during event collection on high-volume instances by reducing the memory footprint of the fetch flow.
- Fixed an issue where an expired Akamai offset (older than 12 hours, HTTP 416) caused the fetch run to fail. The integration now automatically recovers within the same run and restarts fetching from the latest window Akamai still accepts.
- Updated the default value and description of the *Fetch limit* and *Akamai Page size* parameters.
