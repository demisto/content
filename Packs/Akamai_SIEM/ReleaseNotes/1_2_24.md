
#### Integrations

##### Akamai WAF SIEM

- Improved the memory footprint of the event collection flow to reduce peak memory usage during fetch, lowering the chance of out-of-memory failures on high-volume instances.
- Improved the handling of expired Akamai offsets (older than 12 hours, HTTP 416). The integration now automatically recovers within the same fetch run and restarts fetching from the latest window Akamai still accepts, instead of failing the run.
- Updated the default value and description of the *Fetch limit* and *Akamai Page size* parameters.
